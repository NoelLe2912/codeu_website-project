Chapter 3: Functions

Building Your Own Reusable Code Blocks

Define Functions Use Parameters Create Loops

Creating Your Own Functions

So far, you've been using functions that Python provides, like print and len. Now you'll learn to write your own! A function definition tells Python how to perform a specific task that you can use over and over.

Anatomy of a Function

A function definition starts with the keyword def, followed by the function name, parentheses, and a colon. The code inside (the body) must be indented. 

def greet_user():
    print("Welcome to Python!")
    print("Let's learn about functions.")

Header

The first line with def, the name, and the colon

def greet_user():

Body

The indented statements that run when called

print("Welcome!")

Calling Your Function

After you define a function, you can run it by writing its name followed by parentheses: 

# Call the function
greet_user()

# Output:
# Welcome to Python!
# Let's learn about functions.

💡 Key Point: Defining a function doesn't run it—it just creates it. You must call the function to execute the code inside.

Adding Parameters

Parameters make functions flexible by allowing you to pass different values each time you call them. A parameter is like a placeholder that gets filled in when the function runs.

Functions with One Parameter

The parameter name goes inside the parentheses in the function header: 

def display_message(text):
    print(text)
    print(text)

# Call it with different arguments
display_message("Hello!")
display_message("Python is fun")

How Parameters Work

When you call a function with an argument, Python assigns that value to the parameter. It's like creating a temporary variable inside the function. 

# These two are equivalent:
display_message("Hi there")

# Is the same as:
text = "Hi there"
print(text)
print(text)

Using Variables as Arguments

You can pass variables to functions, not just literal values: 

message = "Good morning!"
display_message(message)

# The value of message gets assigned to text

Functions Working Together

One of the most powerful features of functions is that they can call other functions. This lets you build complex programs from simple, reusable pieces.

Building Blocks

Let's create a simple example with multiple functions that work together: 

# A function with two parameters
def echo(word, count):
    print(word * count)

# A function that uses echo
def show_pattern():
    echo("*", 5)
    echo("-", 3)
    echo("*", 5)

# Call the top-level function
show_pattern()
# Output:
# *****
# ---
# *****

Function Call Chain

When you run show_pattern(), it calls echo three times. Each call to echo then calls print. That's a chain of function calls!

show_pattern()
↓
echo("*", 5)
↓
print("*****")

✓ Design Tip: Breaking your program into smaller functions makes it easier to understand, test, and fix. Each function should do one clear task.

Repeating with For Loops

The for loop lets you repeat actions without writing the same code over and over. It's one of the most useful tools in programming.

Basic For Loop Structure

A for loop has a header (ending with a colon) and an indented body: 

for num in range(3):
    print(num)

# Output:
# 0
# 1
# 2

How It Works

Step 1: The range(3) creates a sequence: 0, 1, 2

Step 2: Python assigns the first value (0) to num and runs the body

Step 3: It loops back, assigns the next value (1) to num, and runs the body again

Step 4: This continues until all values are used

Loops in Functions

You can put loops inside functions to repeat actions a specific number of times: 

def display_countdown(start):
    for i in range(start):
        print("T-minus", start - i)
    print("Liftoff!")

display_countdown(5)
# Output:
# T-minus 5
# T-minus 4
# T-minus 3
# T-minus 2
# T-minus 1
# Liftoff!

⚡ Remember: Python counts starting from 0. So range(5) gives you 0, 1, 2, 3, 4—that's five numbers total, but the highest is 4.

Variable Scope: Local vs Global

Variables created inside a function are local—they only exist while that function is running. Once the function finishes, those variables disappear.

Local Variables Example

def combine_and_display(first, second):
    combined = first + second  # Local variable
    print(combined)
    print(combined)

word1 = "Hello "
word2 = "World"
combine_and_display(word1, word2)
# Output:
# Hello World
# Hello World

# But this causes an error:
print(combined)  # NameError: combined doesn't exist here!

Inside the Function

Variables and parameters are accessible and usable

combined ✓
first ✓
second ✓

Outside the Function

Local variables and parameters don't exist

combined ✗
first ✗
second ✗

📚 Why This Matters: Local variables let different functions use the same variable names without conflicts. Each function has its own separate workspace.

Visualizing Function Calls: Stack Diagrams

Stack diagrams help you understand which variables belong to which function. They're especially useful when functions call other functions.

What Are Frames?

Each function call creates a frame—a box containing that function's parameters and local variables. Frames stack on top of each other.

print()

Built-in function

display_message()

text → "Hello"

__main__

message → "Hello"

Reading Stack Diagrams

• The bottom frame is __main__ (your main program)

• Each function call adds a new frame on top

• When a function finishes, its frame is removed

• Variables in each frame are separate from other frames

Understanding Tracebacks

When an error occurs inside a function, Python shows you a traceback—a report of which functions were running when the error happened.

Reading Tracebacks

A traceback shows the chain of function calls from your main code down to where the error occurred: 

Traceback (most recent call last):
  File "main.py", line 10, in <module>
    combine_and_display(word1, word2)
  File "main.py", line 5, in combine_and_display
    print(undefined_var)
NameError: name 'undefined_var' is not defined

How to Read It

Top: Shows where you called the function

Middle: Shows the chain of function calls

Bottom: Shows exactly where the error occurred and what went wrong

🔍 Debugging Tip: Start reading from the bottom of the traceback—that's where the actual error is. The lines above show how you got there.

The Power of Functions

You might wonder why you should bother creating functions when you could just write the code directly. Here are the compelling reasons:

📛

Organization

Give meaningful names to blocks of code, making programs easier to read and understand

🔄

Reusability

Write code once, use it many times. No need to copy and paste the same code

🔧

Maintainability

Fix bugs or make improvements in one place, and the changes apply everywhere

🧩

Modularity

Break complex problems into smaller, manageable pieces you can test independently

Example: Without vs With Functions

Without Functions (Repetitive)

print("="*20)
print("Menu")
print("="*20)
# ... 50 more lines ...
print("="*20)
print("Settings")
print("="*20)

With Functions (Reusable)

def print_header(title):
    print("="*20)
    print(title)
    print("="*20)

print_header("Menu")
print_header("Settings")

Debugging Functions

Debugging is like being a detective—you gather clues, form hypotheses, and test them. It's a crucial skill that gets easier with practice.

The Debugging Mindset

🔍 Detective Work

Look at the clues (error messages, unexpected output) and figure out what caused them

🧪 Experimentation

Make a hypothesis about what's wrong, change your code, and test if you're right

🪜 Small Steps

Write a little code, test it, then write more. Don't write 100 lines before testing!

Common Function Errors

Forgetting to call the function

greet_user # Missing parentheses!

Wrong number of arguments

echo("Hi") # echo needs 2 arguments!

Incorrect indentation

def test():
print("No indent!") # IndentationError

💪 Pro Tip: If debugging takes a long time, you're probably writing too much code before testing. Test small pieces frequently to catch errors early!
