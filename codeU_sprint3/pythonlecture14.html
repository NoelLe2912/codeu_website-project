Chapter 14: Classes and Functions

Introduction to Object-Oriented Programming

Classes Objects Attributes

Programmer-Defined Types

Python provides built-in types like int, str, and list. Now we'll create our own custom types using classes. A class is like a blueprint for creating objects.

Defining a Class

Create a new type to represent a time of day: 

class Time:
    """Represents a time of day."""

# The class is like a factory
type(Time)
# type

# Creating an instance
lunch = Time()
type(lunch)
# __main__.Time

Key Concepts

Class: A programmer-defined type (the blueprint)

Class object: The result of a class definition

Instance: An object created from a class

Instantiation: The process of creating an instance

💡 Think of it: A class is like a cookie cutter, and instances are the cookies you make with it!

Attributes: Object Variables

Attributes are variables that belong to an object. They store the object's data and define its state. Use dot notation to create and access attributes.

Creating Attributes

Add data to an object using the dot operator: 

lunch = Time()

# Create attributes
lunch.hour = 11
lunch.minute = 59
lunch.second = 1

# Access attributes
lunch.hour
# 11

# Use in expressions
total_minutes = lunch.hour * 60 + lunch.minute
total_minutes
# 719

Formatting with F-Strings

Display time with proper formatting using format specifiers: 

# Without formatting
f'{lunch.hour}:{lunch.minute}:{lunch.second}'
# '11:59:1'  (wrong!)

# With format specifiers (02d = 2 digits, leading zero)
f'{lunch.hour:02d}:{lunch.minute:02d}:{lunch.second:02d}'
# '11:59:01'  (correct!)

Functions with Objects

Pass objects as function arguments: 

def print_time(time):
    s = f'{time.hour:02d}:{time.minute:02d}:{time.second:02d}'
    print(s)

print_time(lunch)
# 11:59:01

📝 Pronunciation: AT-trib-ute (emphasis on first syllable), not a-TRIB-ute!

Objects as Return Values

Functions can create and return objects, making it easy to build new instances with specific values.

Factory Function Pattern

Create a function that builds and returns objects: 

def make_time(hour, minute, second):
    time = Time()
    time.hour = hour
    time.minute = minute
    time.second = second
    return time

# Use it to create objects
time = make_time(11, 59, 1)
print_time(time)
# 11:59:01

Why This Pattern?

• Cleaner than creating objects manually

• Ensures all attributes are initialized

• Parameters can have the same names as attributes (common pattern)

• Makes code more readable and maintainable

🏭 Factory Pattern: This is like a factory that builds objects to your specifications!

Objects Are Mutable

Unlike strings and tuples, objects created from custom classes are mutable—you can change their attributes after creation. This power requires care!

Modifying Objects

Change attributes directly: 

start = make_time(9, 20, 0)
print_time(start)
# 09:20:00

# Modify the object
start.hour += 1
start.minute += 32

print_time(start)
# 10:52:00

Impure Functions

Functions that modify their parameters: 

def increment_time(time, hours, minutes, seconds):
    time.hour += hours
    time.minute += minutes
    time.second += seconds

start = make_time(9, 20, 0)
increment_time(start, 1, 32, 0)
print_time(start)
# 10:52:00  (start was modified!)

The Aliasing Problem

Inside the function, the parameter is an alias for the argument:

• time and start refer to the same object

• Changes to time affect start

• The variable name start becomes misleading (it's now the end time!)

⚠️ Side Effects: Functions that modify objects create side effects that can be confusing. Use with care!

Copying Objects

To avoid unwanted modifications, we can copy objects. The copy module provides a function that duplicates any object.

Making Copies

Create independent duplicates: 

from copy import copy

start = make_time(9, 20, 0)
end = copy(start)

# Same data
print_time(start)
print_time(end)
# 09:20:00
# 09:20:00

# But different objects
start is end
# False

The == Operator Surprise

For custom classes, == checks identity by default: 

start == end
# False (even though they have the same data!)

# For custom classes:
# == behaves like 'is' (checks identity)
# Not like built-in types (which check equivalence)

💡 Remember: Copying creates a new object with the same data. Changes to the copy don't affect the original!

Pure Functions

Pure functions don't modify their parameters—they create and return new objects instead. This makes code easier to understand and debug.

Pure Function Example

Use copy to avoid modifying the original: 

def add_time(time, hours, minutes, seconds):
    total = copy(time)
    increment_time(total, hours, minutes, seconds)
    return total

start = make_time(9, 20, 0)
end = add_time(start, 1, 32, 0)

print_time(end)
# 10:52:00

print_time(start)
# 09:20:00  (unchanged!)

Impure Functions

• Modify parameters

• Have side effects

• Can be more efficient

• Harder to debug

• Can cause confusion

Pure Functions

• Don't modify parameters

• No side effects

• Easier to understand

• Easier to test

• More predictable

📚 Functional Style: Write pure functions when reasonable. Use impure functions only when there's a compelling advantage.

Prototype and Patch Development

Start with a simple version that works for easy cases, then fix bugs as you discover them. This is effective but can lead to complicated code.

The Problem

Our increment_time function breaks with certain inputs: 

start = make_time(9, 40, 0)
end = add_time(start, 1, 32, 0)
print_time(end)
# 10:72:00  (Invalid! Minutes can't be 72!)

Patch: Handle Overflow

Fix it by checking for values >= 60: 

def increment_time(time, hours, minutes, seconds):
    time.hour += hours
    time.minute += minutes
    time.second += seconds

    if time.second >= 60:
        time.second -= 60
        time.minute += 1

    if time.minute >= 60:
        time.minute -= 60
        time.hour += 1

Better Patch: Use divmod

Handle any size increment: 

def increment_time(time, hours, minutes, seconds):
    time.hour += hours
    time.minute += minutes
    time.second += seconds
    
    carry, time.second = divmod(time.second, 60)
    carry, time.minute = divmod(time.minute + carry, 60)
    carry, time.hour = divmod(time.hour + carry, 24)

⚠️ Limitation: Prototype and patch works but can create messy code with many special cases. Better to plan ahead!

Design-First Development

Think deeply about the problem before coding. A good insight can make everything simpler and more elegant.

The Key Insight

Time is like a three-digit number in base 60:

• Second: "ones column" (0-59)

• Minute: "sixties column" (0-59)

• Hour: "thirty-six hundreds column" (0-23)

• Solution: Convert to integers, use arithmetic, convert back!

Conversion Functions

Convert between Time objects and integers: 

def time_to_int(time):
    minutes = time.hour * 60 + time.minute
    seconds = minutes * 60 + time.second
    return seconds

def int_to_time(seconds):
    minute, second = divmod(seconds, 60)
    hour, minute = divmod(minute, 60)
    return make_time(hour, minute, second)

# Test it
time = make_time(1, 1, 1)
time_to_int(time)
# 3661 seconds since midnight

Elegant add_time

Much simpler with conversions: 

def add_time(time, hours, minutes, seconds):
    duration = make_time(hours, minutes, seconds)
    seconds = time_to_int(time) + time_to_int(duration)
    return int_to_time(seconds)

start = make_time(9, 40, 0)
end = add_time(start, 1, 32, 0)
print_time(end)
# 11:12:00  (Works perfectly!)

✨ Paradox: Sometimes making a problem more general (or harder) makes the solution easier! Fewer special cases = fewer bugs.

Debugging with Objects

Python provides built-in functions for inspecting objects and their attributes.

Inspection Functions

# Check object type
type(start)
# __main__.Time

# Check if instance of a class
isinstance(start, Time)
# True

# Check if attribute exists
hasattr(start, 'hour')
# True

# Get all attributes as dictionary
vars(start)
# {'hour': 9, 'minute': 20, 'second': 0}

🔍 Pro Tip: Use vars() to quickly see all attributes and their values when debugging!
