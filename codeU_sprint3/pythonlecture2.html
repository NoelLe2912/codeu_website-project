Creating Variables

A variable is like a labeled container that holds a value. You can think of it as giving a name to a piece of data so you can use it later. Creating a variable is called assignment, and it uses the equals sign (=).

Basic Assignment

The assignment statement has three parts: the variable name (left), the equals sign (middle), and the value (right).

Integer Assignment

count = 25

Float Assignment

temperature = 98.6

String Assignment

greeting = 'Hello, Python!'

Using Variables

After you create a variable, you can use it anywhere you would use a value. The variable acts as a substitute for its contents. 

# Create a variable
age = 21

# Use it in calculations
age + 4        # Result: 25

# Use it with functions
round(age / 3) # Result: 7

💡 Important: When you run an assignment statement, nothing appears on screen. Python creates the variable silently. To see what's stored in a variable, type its name or use the print function.

Visualizing Variables: State Diagrams

A state diagram is a visual representation showing what values your variables hold at a specific moment. It's like taking a snapshot of your program's memory.

State Diagram Example

count → 25
temperature → 98.6
greeting → 'Hello, Python!'

📊 Why State Diagrams Matter: They help you track what your variables contain as your program runs. This becomes especially useful when debugging or understanding complex programs.

Naming Your Variables

Variable names aren't completely freeform. Python has specific rules about what makes a valid name, and following conventions makes your code easier to read.

The Rules

✓ Can contain letters, numbers, and underscores (_)

✓ Must start with a letter or underscore

✓ Can be any length

→ Uppercase is allowed but lowercase is conventional

✗ Cannot start with a number

✗ Cannot contain spaces or punctuation (except underscore)

✗ Cannot be a Python keyword

✓ Valid Names

user_name
total_count
item2
_private_var
averageScore

✗ Invalid Names

2fast        # starts with number
user-name    # contains dash
my var       # contains space
class        # keyword
item!        # contains punctuation

Importing Modules

Python comes with many built-in collections of useful code called modules. To access the functions and values in a module, you need to import it first.

The math Module

One of the most commonly used modules is math, which provides mathematical functions and constants. 

# First, import the module
import math

# Now you can access its contents
math.pi           # Result: 3.141592653589793
math.sqrt(16)     # Result: 4.0
math.pow(3, 4)    # Result: 81.0

The Dot Operator

To use something from a module, write the module name, then a dot (.), then the name of the variable or function. This tells Python exactly where to find what you're looking for. 

math.sqrt(100)    # sqrt function from math module
math.e            # e constant from math module

⚡ Power Tip: When you see code like math.pow(2, 3), read it as "the pow function from the math module." The dot connects the module to its contents.

Expressions vs Statements

Understanding the difference between expressions and statements is fundamental to reading and writing Python code.

Expressions

An expression is anything that has a value. When Python evaluates an expression, it produces a result. 

42
7 + 5
math.sqrt(25)
user_name

Statements

A statement is a command that does something. When Python executes a statement, it performs an action. 

x = 10
import math
print(x)

Key Differences

Expressions have values that can be used in calculations

Statements perform actions but don't produce values

Example: In total = 15 + 7, the part 15 + 7 is an expression (value: 22), and the whole line is a statement (action: assign)

Displaying Output with print

The print function is how you show information to users. Unlike just typing an expression (which only shows the last result), print can display multiple values.

Why Use print?

When you type expressions in Python, only the last one gets displayed: 

# Only the second value appears
count + 5
count + 10    # Only this shows

But with print, you can show as many values as you want: 

print(count + 5)
print(count + 10)  # Both appear now!

Multiple Arguments

You can pass multiple values to print separated by commas. Python will display them all with spaces in between. 

name = 'Alice'
score = 95

print('Student:', name, 'Score:', score)
# Output: Student: Alice Score: 95

✓ Pro Tip: print automatically adds spaces between values and moves to a new line after printing. This makes it perfect for displaying formatted output!

Understanding Function Arguments

When you call a function, the values you put in the parentheses are called arguments. Different functions accept different numbers and types of arguments.

Required Arguments

One Argument

abs(-42) # Result: 42

The abs function needs exactly one number

Two Arguments

math.pow(4, 3) # Result: 64.0

The pow function requires a base and an exponent

Variable Arguments

print('Any', 'number', 'of', 'values')

The print function accepts any number of arguments

Optional Arguments

Some functions have optional arguments that provide additional control: 

# round with one argument
round(3.7)           # Result: 4

# round with optional second argument (decimal places)
round(3.14159, 2)    # Result: 3.14

# int with optional base argument
int('1010', 2)       # Result: 10 (binary to decimal)

⚠️ Common Errors:

• Too many arguments: abs(5, 2) ← TypeError

• Too few arguments: math.pow(4) ← TypeError

• Wrong type: math.sqrt('nine') ← TypeError

Writing Comments

Comments are notes to yourself and other programmers. They explain what your code does in plain English, making it easier to understand and maintain.

How to Write Comments

Any text after a # symbol is ignored by Python. Comments can appear on their own line or at the end of a code line. 

# Calculate total minutes in a day
minutes = 24 * 60

hours = 8      # typical work day length

✓ Good Comments

# velocity in meters/second
v = 15

# convert temperature to Celsius
celsius = (fahr - 32) * 5/9

✗ Useless Comments

# assign 15 to v
v = 15

# add two numbers
result = x + y

📝 Comment Best Practices: Explain why you're doing something, not what the code obviously does. Assume readers can read Python—help them understand your reasoning and intent.

Three Types of Errors

Every programmer encounters errors. Knowing what type of error you're dealing with helps you fix it faster.

🔴

Syntax Error

Python can't understand your code

million! = 100

Invalid punctuation

🟡

Runtime Error

Code runs but crashes

'100' / 5

Can't divide string

🟣

Semantic Error

Code runs but wrong result

1 + 5 / 2

Wrong order of ops

Detailed Examples

Syntax Error: Breaks structure rules

student count = 30 # spaces not allowed

Python refuses to run the program

Runtime Error (Exception): Type mismatch

'hello' - 5 # can't subtract from string

Program starts but crashes during execution

Semantic Error: Logic mistake

average = 10 + 20 / 2 # Result: 20, not 15!

Need parentheses: (10 + 20) / 2
