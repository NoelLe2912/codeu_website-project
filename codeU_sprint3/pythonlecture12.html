Chapter 12: Text Analysis and Generation

Analyzing Word Patterns and Generating New Text

Word Frequency Bigrams Markov Chains

Counting Unique Words

Text analysis begins with understanding what words appear in a text and how often. We'll use dictionaries to track unique words and their frequencies—the foundation of natural language processing.

Basic Word Counting

Use a dictionary to track unique words: 

text = "the quick brown fox jumps over the lazy dog"

unique_words = {}
for word in text.split():
    unique_words[word] = 1

len(unique_words)
# 8 (unique words)

# The words are the keys
list(unique_words.keys())
# ['the', 'quick', 'brown', 'fox', 'jumps', 'over', 'lazy', 'dog']

Finding Longest Words

Sort words by length using a key function: 

# Sort by length
sorted(unique_words, key=len)
# ['the', 'fox', 'dog', 'over', 'lazy', 'quick', 'brown', 'jumps']

# Get the 3 longest words
sorted(unique_words, key=len)[-3:]
# ['quick', 'brown', 'jumps']

💡 Key Insight: Using dictionary keys to store unique items is a common pattern. The values don't matter—we're just using the dictionary to track what we've seen.

Cleaning Text Data

Real text contains punctuation, capitalization, and other noise. We need to clean it before analysis to get accurate results.

Removing Punctuation

Use strip() to remove punctuation from word boundaries: 

punctuation = '.!?,;:\'"'

def clean_word(word):
    return word.strip(punctuation).lower()

# Test it
clean_word('"Hello!"')
# 'hello'

clean_word('world.')
# 'world'

Processing Complete Text

Apply cleaning to all words in a text: 

text = "Hello, world! How are you? I'm fine, thanks."

unique_words = {}
for word in text.split():
    word = clean_word(word)
    if word:  # Skip empty strings
        unique_words[word] = 1

list(unique_words.keys())
# ['hello', 'world', 'how', 'are', 'you', "i'm", 'fine', 'thanks']

🧹 Important: Always clean and normalize text before analysis. Convert to lowercase and remove punctuation for consistent results.

Counting Word Frequencies

Instead of just tracking unique words, we can count how many times each word appears. This reveals patterns about which words are most important or common.

Building a Word Counter

Count occurrences using a dictionary: 

text = "the cat in the hat sat on the mat"

word_counter = {}
for word in text.split():
    if word not in word_counter:
        word_counter[word] = 1
    else:
        word_counter[word] += 1

word_counter
# {'the': 3, 'cat': 1, 'in': 1, 'hat': 1, 'sat': 1, 'on': 1, 'mat': 1}

Finding Most Common Words

Sort by frequency to find the most common words: 

def second_element(t):
    return t[1]

# Sort items by frequency
items = sorted(word_counter.items(), 
               key=second_element, 
               reverse=True)

# Print top 3
for word, freq in items[:3]:
    print(freq, word)

# Output:
# 3 the
# 1 cat
# 1 in

📊 Pattern: This frequency counting pattern is fundamental to text analysis, search engines, and recommendation systems!

Functions with Optional Parameters

Optional parameters let you provide default values while allowing users to override them when needed. This makes functions more flexible and easier to use.

Defining Optional Parameters

Use = in the parameter list to set default values: 

def print_most_common(word_counter, num=5):
    items = sorted(word_counter.items(), 
                   key=lambda t: t[1], 
                   reverse=True)
    
    for word, freq in items[:num]:
        print(freq, word)

# Use default (5)
print_most_common(word_counter)

# Override with 3
print_most_common(word_counter, 3)

Rules for Optional Parameters

• Required parameters must come first

• Optional parameters follow required ones

• Default values are assigned during function definition

• Callers can override defaults by providing arguments

✨ Best Practice: Use optional parameters to make common use cases simple while keeping flexibility for advanced users.

Dictionary Subtraction

Sometimes you need to find elements in one set that aren't in another. Dictionary subtraction lets you compare two dictionaries and find the differences.

The Subtract Function

Find keys in d1 that are not in d2: 

def subtract(d1, d2):
    res = {}
    for key in d1:
        if key not in d2:
            res[key] = d1[key]
    return res

# Example: spell checking
book_words = {'the': 100, 'cat': 5, 'happyness': 1}
valid_words = {'the': 1, 'cat': 1, 'happiness': 1}

misspelled = subtract(book_words, valid_words)
misspelled
# {'happyness': 1}

Finding Rare Words

Filter words that appear only once: 

singletons = []
for word, freq in word_counter.items():
    if freq == 1:
        singletons.append(word)

print(singletons)
# ['cat', 'in', 'hat', 'sat', 'on', 'mat']

🎯 Use Case: Dictionary subtraction is perfect for spell-checking, finding unique items, or comparing datasets.

Analyzing Word Pairs (Bigrams)

A bigram is a sequence of two consecutive words. Analyzing bigrams reveals patterns about which words commonly appear together—the foundation of predictive text and language models.

What Are Bigrams?

From the text "the cat sat", we get these bigrams:

• ("the", "cat")

• ("cat", "sat")

Sequences of three words are called trigrams, and sequences of n words are called n-grams.

Counting Bigrams

Use a sliding window to extract consecutive word pairs: 

text = "the cat in the hat"
words = text.split()

bigram_counter = {}
window = []

for word in words:
    window.append(word)
    
    if len(window) == 2:
        key = tuple(window)
        if key not in bigram_counter:
            bigram_counter[key] = 1
        else:
            bigram_counter[key] += 1
        window.pop(0)

bigram_counter
# {('the', 'cat'): 1, ('cat', 'in'): 1, 
#  ('in', 'the'): 1, ('the', 'hat'): 1}

Most Common Bigrams

Find frequently occurring word pairs: 

def second_element(t):
    return t[1]

items = sorted(bigram_counter.items(), 
               key=second_element, 
               reverse=True)

for bigram, freq in items[:3]:
    print(freq, bigram)

💡 Foundation: Bigrams are the building blocks of language models. Modern AI systems like ChatGPT use similar (but more sophisticated) techniques!

Markov Chain Analysis

Markov analysis tracks which words can follow each word. This creates a map of possible word sequences that we can use to generate new, similar text.

Building a Successor Map

Map each word to the words that can follow it: 

text = "the cat sat on the mat the cat ran"
words = text.split()

successor_map = {}
window = []

for word in words:
    window.append(word)
    
    if len(window) == 2:
        first, second = window
        
        if first not in successor_map:
            successor_map[first] = [second]
        else:
            successor_map[first].append(second)
        
        window.pop(0)

successor_map
# {'the': ['cat', 'mat', 'cat'], 
#  'cat': ['sat', 'ran'], 
#  'sat': ['on'], 
#  'on': ['the'], 
#  'mat': ['the']}

Understanding the Map

Each word points to its possible successors:

• "the" can be followed by "cat" (appears twice) or "mat" (once)

• "cat" can be followed by "sat" or "ran"

• Words appearing multiple times have higher probability

🔗 Key Idea: Duplicates in the successor list represent probability—"cat" appears twice after "the", so it's twice as likely to be chosen.

Generating New Text

Using the successor map, we can generate new text that follows the same patterns as the original. This is a simple version of how predictive text and AI language models work!

The Generation Algorithm

Start with a word, then repeatedly choose successors: 

import random

# Start with a word
word = 'the'
result = [word]

# Generate 10 more words
for i in range(10):
    if word in successor_map:
        # Get possible successors
        successors = successor_map[word]
        # Choose one randomly
        word = random.choice(successors)
        result.append(word)
    else:
        break

print(' '.join(result))
# Example output: "the cat sat on the mat the cat ran"

Why It Works

Step 1: Start with any word from the original text

Step 2: Look up which words can follow it

Step 3: Choose one randomly (common successors chosen more often)

Step 4: Repeat with the chosen word

Improving Results

Better results with bigrams as keys (trigram analysis):

• Instead of mapping single words to successors

• Map word pairs (bigrams) to their successors

• This preserves more context and generates better text

• Example: ("the", "cat") → ["sat", "ran"]

🤖 Real-World Connection: This is essentially how early chatbots worked! Modern LLMs use the same core idea but with billions of parameters and much more context.

Debugging Complex Programs

As programs get larger and more complex, debugging becomes more challenging. Here are systematic strategies for tackling difficult bugs.

The Five Rs of Debugging

📖 Reading

Examine your code carefully. Does it say what you meant?

▶️ Running

Experiment with changes. Display values at key points.

💭 Ruminating

Think about error types and what could cause them.

🦆 Rubberducking

Explain the problem aloud—often you'll find the answer!

⬅️ Retreating

Back up to working code, then rebuild carefully.

Rubber Duck Debugging

This is a real technique! Explain your problem step-by-step to an inanimate object (like a rubber duck). Often, the act of explaining reveals the solution.

💡 Balance: Don't get stuck on one approach. If reading code isn't working, try running experiments. If experiments fail, step back and think.
