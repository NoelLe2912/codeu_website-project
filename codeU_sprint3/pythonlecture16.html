Chapter 16: Classes and Objects

Continuing our tour of Object-Oriented Programming (OOP)

Identity Deep Copy Polymorphism

Creating a Point

In computer graphics, a location on the screen is often represented using a pair of coordinates in an x-y plane. By convention, the point (0, 0) usually represents the upper-left corner of the screen, and (x, y) represents the point x units to the right and y units down from the origin.

Defining the Point Class

In object-oriented programming, it is most idiomatic to create a new type to represent points as objects. 

class Point:
    """Represents a point in 2-D space."""

    def __init__(self, x, y):
        self.x = x
        self.y = y
        
    def __str__(self):
        return f'Point({self.x}, {self.y})'

# Instantiate and display a Point object
start = Point(0, 0)
print(start)
# Output: Point(0, 0)

Translating a Point

We can write a method like translate that modifies an existing point, or a method like translated that creates a new one. 

# Inside class Point...
    def translate(self, dx, dy):
        self.x += dx
        self.y += dy

    def translated(self, dx=0, dy=0):
        from copy import copy
        point = copy(self)
        point.translate(dx, dy)
        return point

end2 = start.translated(0, 150)
print(end2)
# Output: Point(0, 150)

Creating a Line

Now let's define a class that represents the line segment between two points. As usual, we'll start with an __init__ method and a __str__ method. 

class Line:
    def __init__(self, p1, p2):
        self.p1 = p1
        self.p2 = p2
        
    def __str__(self):
        return f'Line({self.p1}, {self.p2})'

# Instantiate using Point objects
end1 = Point(300, 0)
line1 = Line(start, end1)
print(line1)
# Output: Line(Point(0, 0), Point(300, 0))

💡 Nested Objects: Notice how the f-string in __str__ evaluates self.p1. Because the result is a Point object, it automatically invokes the __str__ method from the Point class!

Equivalence and Identity

Suppose we create two points with the exact same coordinates. Are they the same object? 

p1 = Point(200, 100)
p2 = Point(200, 100)

p1 == p2
# False (by default, == checks if they are the exact same object)

p1 is p2
# False

Overriding the == Operator

If we want to change that behavior, we can provide a special method called __eq__ that defines what it means for two Point objects to be "equal". 

# Inside class Point...
    def __eq__(self, other):
        return (self.x == other.x) and (self.y == other.y)

# Now it checks equivalence!
p1 == p2
# True

⚠️ Remember: It's not possible to override the is operator—it always checks whether the objects are identical in memory. But for your types, you can override == to check equivalence.

Rectangles & The Deep Copy Problem

Let's define a Rectangle specified by its width, height, and the location of its upper-left corner (a Point). 

class Rectangle:
    def __init__(self, width, height, corner):
        self.width = width
        self.height = height
        self.corner = corner
        
    def __str__(self):
        return f'Rectangle({self.width}, {self.height}, {self.corner})'

    def translate(self, dx, dy):
        self.corner.translate(dx, dy)

The Aliasing Bug

When we use standard copy() to duplicate a rectangle, it copies the Rectangle object but not the nested Point object. They end up sharing the same corner! 

from copy import copy

box1 = Rectangle(100, 50, Point(30, 20))
box2 = copy(box1)

box1 is box2
# False (they are different rectangles)

box1.corner is box2.corner
# True! (they share the SAME corner object)

box2.translate(30, 20) 
# Wait, this just moved box1's corner too!

The Fix: deepcopy

The copy module provides deepcopy, which copies not only the object but also the objects it refers to. 

from copy import deepcopy

box3 = Rectangle(100, 50, Point(20, 20))
box4 = deepcopy(box3)

box3.corner is box4.corner
# False (Now they are completely separate!)

Polymorphism

If we give Line, Rectangle, and a custom Circle class their own draw() methods, we can do something powerful: 

shapes = [line1, line2, box3, box4]

for shape in shapes:
    shape.draw()

✨ Polymorphism: The word comes from Greek roots that mean "many shaped". It's the ability of different types to provide the same methods, allowing you to perform operations by invoking the same method on different types of objects. In a sense, each object knows how to draw itself.

Debugging & Glossary

Avoiding Bugs

In general, there are two ways to avoid aliasing bugs like the one we saw with the shared Point:

  • Avoid sharing objects: Use deepcopy when duplicating nested structures.
  • Avoid modifying objects: Replace impure functions (like translate) with pure functions (like translated) that return new objects instead of altering existing ones.

Glossary

shallow copy: A copy operation that does not copy nested objects.

deep copy: A copy operation that also copies nested objects.

polymorphism: The ability of a method or operator to work with multiple types of objects.
