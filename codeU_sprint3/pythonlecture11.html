Chapter 11: Tuples

Immutable Sequences for Efficient Data Handling

Immutable Hashable Pack/Unpack

Tuples Are Like Lists

A tuple is a sequence of values, just like a list. Values can be any type and are indexed by integers. The key difference? Tuples are immutable—once created, they cannot be changed.

Creating Tuples

Use commas to create tuples (parentheses are optional but common): 

# Without parentheses
t = 'l', 'u', 'p', 'i', 'n'
type(t)
# tuple

# With parentheses (more common)
t = ('l', 'u', 'p', 'i', 'n')
type(t)
# tuple

Single-Element Tuples

You must include a trailing comma for single-element tuples: 

# With comma - this is a tuple
t1 = 'p',
type(t1)
# tuple

# Without comma - this is just a string
t2 = ('p')
type(t2)
# str

Using the tuple() Function

Convert sequences to tuples or create empty tuples: 

# Empty tuple
t = tuple()
t
# ()

# Convert string to tuple
t = tuple('lupin')
t
# ('l', 'u', 'p', 'i', 'n')

List Operations Work on Tuples

Most list operators work with tuples too: 

t = ('l', 'u', 'p', 'i', 'n')

# Indexing
t[0]
# 'l'

# Slicing
t[1:3]
# ('u', 'p')

# Concatenation
tuple('lup') + ('i', 'n')
# ('l', 'u', 'p', 'i', 'n')

# Repetition
tuple('spam') * 2
# ('s', 'p', 'a', 'm', 's', 'p', 'a', 'm')

📝 Note: Avoid using "tuple" as a variable name—it's the name of a built-in function!

But Tuples Are Immutable

The critical difference between tuples and lists: you cannot modify tuple elements. This immutability makes tuples hashable and safe to use as dictionary keys.

Cannot Modify Elements

Attempting to change a tuple causes a TypeError: 

t = ('l', 'u', 'p', 'i', 'n')
t[0] = 'L'
# TypeError: 'tuple' object does not support item assignment

No Modifying Methods

Tuples don't have methods like append() or remove(): 

t.remove('l')
# AttributeError: 'tuple' object has no attribute 'remove'

Tuples as Dictionary Keys

Because tuples are immutable, they're hashable and can be used as dictionary keys: 

d = {}
d[1, 2] = 3
d[3, 4] = 7

# Look up using a tuple
d[1, 2]
# 3

# Or use a variable
t = (3, 4)
d[t]
# 7

Tuples as Dictionary Values

Tuples can also be values in dictionaries: 

t = tuple('abc')
d = {'key': t}
d
# {'key': ('a', 'b', 'c')}

✓ Lists: Mutable

• Can change elements

• Not hashable

• Cannot be dict keys

✓ Tuples: Immutable

• Cannot change elements

• Hashable

• Can be dict keys

Tuple Assignment

Python allows you to assign multiple values to multiple variables in a single statement. This elegant feature makes many operations simpler and more readable.

Basic Tuple Assignment

Assign multiple values at once: 

a, b = 1, 2

# Values are assigned left to right
a, b
# (1, 2)

Works with Any Sequence

The right side can be any sequence—list, tuple, or string: 

# Split an email address
email = 'monty@python.org'
username, domain = email.split('@')

username, domain
# ('monty', 'python.org')

Must Match Length

The number of variables must equal the number of values: 

a, b = 1, 2, 3
# ValueError: too many values to unpack (expected 2)

Swapping Values

Swap variables without a temporary variable: 

# Traditional way with temp variable
temp = a
a = b
b = temp

# Elegant way with tuple assignment
a, b = b, a

Tuple Assignment in Loops

Use with items() to loop through dictionaries elegantly: 

d = {'one': 1, 'two': 2}

# Concise version
for key, value in d.items():
    print(key, '->', value)

# Output:
# one -> 1
# two -> 2

⚡ Elegant Pattern: Tuple assignment makes code cleaner and more Pythonic. Use it whenever you're unpacking sequences!

Tuples as Return Values

Functions can only return one value—but if that value is a tuple, you can effectively return multiple values. This is a powerful and common pattern.

Built-in Example: divmod()

divmod() returns both quotient and remainder as a tuple: 

divmod(7, 3)
# (2, 1)

# Use tuple assignment to unpack
quotient, remainder = divmod(7, 3)
quotient
# 2
remainder
# 1

Custom Function Example

Return multiple values from your own functions: 

def min_max(t):
    return min(t), max(t)

# Returns a tuple
min_max([2, 4, 1, 3])
# (1, 4)

# Unpack the results
low, high = min_max([2, 4, 1, 3])
low, high
# (1, 4)

🎯 Use Case: Returning tuples is perfect when you need to compute multiple related values together—like coordinates, statistics, or results from complex calculations.

Argument Packing and Unpacking

The * operator lets you write functions that accept any number of arguments, and it lets you pass sequences as separate arguments. This creates flexible, powerful functions.

Packing: Variable Arguments

Use *args to accept any number of arguments: 

def mean(*args):
    return sum(args) / len(args)

# Call with any number of arguments
mean(1, 2, 3)
# 2.0

mean(1, 2, 3, 4, 5)
# 3.0

Unpacking: Sequence to Arguments

Use * to unpack a sequence into separate arguments: 

t = (7, 3)

# Without unpacking - error!
divmod(t)
# TypeError: divmod expected 2 arguments, got 1

# With unpacking - works!
divmod(*t)
# (2, 1)

Practical Example: Trimmed Mean

Combine packing and unpacking for flexible functions: 

def trimmed_mean(*args):
    # Find min and max
    low, high = min(args), max(args)
    
    # Remove them
    trimmed = list(args)
    trimmed.remove(low)
    trimmed.remove(high)
    
    # Compute mean (unpack trimmed list)
    return mean(*trimmed)

mean(1, 2, 3, 10)
# 4.0

trimmed_mean(1, 2, 3, 10)
# 2.5  (removes 1 and 10, averages 2 and 3)

💡 Real-World Use: Trimmed means are used in sports like diving and gymnastics to reduce the impact of outlier scores from individual judges.

Pairing Elements with zip()

The zip() function pairs up elements from multiple sequences, making it easy to process corresponding items together—like the teeth of a zipper coming together.

Basic Zipping

Loop through two sequences simultaneously: 

scores1 = [1, 2, 4, 5, 1, 5, 2]
scores2 = [5, 5, 2, 2, 5, 2, 3]

for pair in zip(scores1, scores2):
    print(pair)

# Output:
# (1, 5)
# (2, 5)
# (4, 2)
# (5, 2)
# (1, 5)
# (5, 2)
# (2, 3)

Unpacking Pairs

Use tuple assignment to work with paired values: 

wins = 0
for team1, team2 in zip(scores1, scores2):
    if team1 > team2:
        wins += 1

wins
# 3

Creating Lists of Tuples

Convert a zip object to a list of tuples: 

t = list(zip(scores1, scores2))
t
# [(1, 5), (2, 5), (4, 2), (5, 2), (1, 5), (5, 2), (2, 3)]

# Get the last game result
t[-1]
# (2, 3)

Creating Dictionaries from Pairs

Combine zip() and dict() to map letters to positions: 

letters = 'abcdefghijklmnopqrstuvwxyz'
numbers = range(len(letters))
letter_map = dict(zip(letters, numbers))

letter_map['a'], letter_map['z']
# (0, 25)

Enumerate for Indices

Use enumerate() when you need both indices and elements: 

for index, element in enumerate('abc'):
    print(index, element)

# Output:
# 0 a
# 1 b
# 2 c

Comparing and Sorting Tuples

Tuples can be compared and sorted. Python compares element by element from left to right, making this useful for custom sorting strategies.

Tuple Comparison

Tuples are compared element by element: 

(0, 1, 2) < (0, 3, 4)
# True (1 < 3, so first tuple is smaller)

# Later elements don't matter if earlier ones differ
(0, 1, 2000000) < (0, 3, 4)
# True

Sorting with Default Behavior

Sort tuples by first element, then second to break ties: 

counter = {'b': 1, 'a': 3, 'n': 2}
items = counter.items()

sorted(items)
# [('a', 3), ('b', 1), ('n', 2)]

Custom Sort Keys

Use a key function to sort by specific criteria: 

def second_element(t):
    return t[1]

# Sort by count (second element)
sorted_items = sorted(items, key=second_element)
sorted_items
# [('b', 1), ('n', 2), ('a', 3)]

# Get most common letter
sorted_items[-1]
# ('a', 3)

Using max() with Key

Find maximum without sorting the entire list: 

# More efficient - no sorting needed
max(items, key=second_element)
# ('a', 3)

⚡ Performance Tip: Use max() or min() with a key function when you only need the extreme value—it's faster than sorting the entire collection.

Inverting a Dictionary

Sometimes you need to flip a dictionary—making values into keys and keys into values. Since values don't have to be unique, the inverted dictionary uses lists to handle duplicates.

The Problem

Dictionary values can repeat, but keys cannot: 

d = {'p': 1, 'a': 1, 'r': 2, 'o': 1, 't': 1}

# If we invert this:
# Multiple letters have count 1
# Need: {1: ['p', 'a', 'o', 't'], 2: ['r']}

The Solution

Map each value to a list of keys with that value: 

def invert_dict(d):
    new = {}
    for key, value in d.items():
        if value not in new:
            new[value] = [key]
        else:
            new[value].append(key)
    return new

d = {'p': 1, 'a': 1, 'r': 2, 'o': 1, 't': 1}
invert_dict(d)
# {1: ['p', 'a', 'o', 't'], 2: ['r']}

🎯 Pattern: This is the first example where dictionary values are lists—a powerful compound data structure you'll use often!

Debugging Compound Data Structures

As data structures get more complex—lists of tuples, dictionaries with tuple keys and list values—debugging becomes harder. The structshape module helps visualize complex structures.

Using structshape

The structshape function summarizes data structure types: 

from structshape import structshape

# Simple list
t = [1, 2, 3]
structshape(t)
# 'list of 3 int'

# List of lists
t2 = [[1,2], [3,4], [5,6]]
structshape(t2)
# 'list of 3 list of 2 int'

# Mixed types
t3 = [1, 2, 3, 4.0, '5', '6', [7], [8], 9]
structshape(t3)
# 'list of (3 int, float, 2 str, 2 list of int, int)'

Complex Structures

structshape handles tuples, dictionaries, and nested combinations: 

# List of tuples
s = 'abc'
lt = list(zip([1, 2, 3], s))
structshape(lt)
# 'list of 3 tuple of (int, str)'

# Dictionary
d = dict(lt)
structshape(d)
# 'dict of 3 int->str'

🔧 Debugging Tip: When working with complex nested structures, use structshape to quickly verify that your data has the expected structure and types.
