Chapter 9: Lists

Working with Collections of Values

Mutable Sequences List Methods References

Lists Are Sequences

Like strings, lists are sequences of values. But unlike strings, where values must be characters, lists can contain any type of value—numbers, strings, or even other lists!

Creating Lists

Use square brackets to create a list with elements: 

# List of integers
numbers = [42, 123]

# List of strings
cheeses = ['Cheddar', 'Edam', 'Gouda']

# Mixed types
t = ['spam', 2.0, 5, [10, 20]]

# Empty list
empty = []

List Properties

• Elements can be any type—and can be mixed

• Lists can contain other lists (nested lists)

• Use len() to get the number of elements

• Empty lists have length 0

len(cheeses)
# 3

len(empty)
# 0

len(t)
# 4 (nested list counts as one element)

Nested Lists

A list within another list is called a nested list: 

t = ['spam', 2.0, 5, [10, 20]]

# The nested list is a single element
len(t)
# 4

# 10 is inside the nested list, not in t
10 in t
# False

# But the nested list itself is in t
[10, 20] in t
# True

💡 Key Difference: Strings can only contain characters. Lists can contain anything—including other lists!

Lists Are Mutable

Unlike strings (which are immutable), lists are mutable—you can change their elements. This is one of the most important differences between lists and strings.

Accessing Elements

Use the bracket operator to read elements (just like strings): 

cheeses = ['Cheddar', 'Edam', 'Gouda']

# First element (index 0)
cheeses[0]
# 'Cheddar'

# Last element (negative index)
cheeses[-1]
# 'Gouda'

Modifying Elements

Use the bracket operator on the left side to change elements: 

numbers = [42, 123]

# Change the second element
numbers[1] = 17

numbers
# [42, 17]

Strings: Immutable

s = 'hello'
s[0] = 'H'
# TypeError!

Cannot change characters

Lists: Mutable

lst = ['h', 'e', 'l', 'l', 'o']
lst[0] = 'H'
# Works!

Can change elements freely

The in Operator

Check if an element exists in a list: 

cheeses = ['Cheddar', 'Edam', 'Gouda']

'Edam' in cheeses
# True

'Wensleydale' in cheeses
# False

🔑 Critical Concept: Mutability means you can modify lists in place. This has important implications for how lists behave when passed to functions or assigned to multiple variables!

Slicing Lists

The slice operator works exactly the same way for lists as it does for strings. You can extract portions of a list or create copies.

Basic Slicing

Use [n:m] to get elements from index n up to (but not including) m: 

letters = ['a', 'b', 'c', 'd']

# Get second and third elements
letters[1:3]
# ['b', 'c']

# From beginning to index 2
letters[:2]
# ['a', 'b']

# From index 2 to end
letters[2:]
# ['c', 'd']

Copying Lists

Two ways to create a copy of a list: 

letters = ['a', 'b', 'c', 'd']

# Method 1: Full slice
copy1 = letters[:]

# Method 2: list() function
copy2 = list(letters)

copy1
# ['a', 'b', 'c', 'd']

copy2
# ['a', 'b', 'c', 'd']

⚠️ Note: Avoid using "list" as a variable name—it's the name of a built-in function. If you assign to it, you'll lose access to the function!

List Operations

Lists support several operators and built-in functions that make working with them convenient and powerful.

Concatenation and Repetition

Use + to join lists and * to repeat them: 

t1 = [1, 2]
t2 = [3, 4]

# Concatenate with +
t1 + t2
# [1, 2, 3, 4]

# Repeat with *
['spam'] * 4
# ['spam', 'spam', 'spam', 'spam']

Built-in Functions

Python provides useful functions for working with lists of numbers: 

numbers = [1, 5, 3, 9, 2]

# Sum all elements
sum(numbers)
# 20

# Find minimum
min(numbers)
# 1

# Find maximum
max(numbers)
# 9

💡 Limitation: Most mathematical operators (-, /, etc.) don't work with lists. Use sum(), min(), and max() for common calculations.

Powerful List Methods

Lists come with many built-in methods that modify the list in place. Understanding these methods is crucial for effective list manipulation.

Adding Elements

Use append to add one element, or extend to add multiple: 

letters = ['a', 'b', 'c', 'd']

# Add single element
letters.append('e')
letters
# ['a', 'b', 'c', 'd', 'e']

# Add multiple elements
letters.extend(['f', 'g'])
letters
# ['a', 'b', 'c', 'd', 'e', 'f', 'g']

Removing Elements

Two ways to remove elements: by index (pop) or by value (remove): 

t = ['a', 'b', 'c']

# Remove by index (returns the element)
t.pop(1)
# 'b'

t
# ['a', 'c']

# Remove by value (returns None)
t = ['a', 'b', 'c']
t.remove('b')
# None (no return value shown)

t
# ['a', 'c']

pop(index)

• Removes element at index

• Returns the removed element

• Use when you know position

remove(value)

• Removes first matching element

• Returns None

• Use when you know value

Error: Element Not Found

Trying to remove a non-existent element causes a ValueError: 

t = ['a', 'b', 'c']
t.remove('d')
# ValueError: list.remove(x): x not in list

Converting Between Lists and Strings

Lists and strings are closely related. Python provides powerful methods to convert between them—essential for text processing.

String to List of Characters

Use list() to break a string into individual characters: 

s = 'spam'
t = list(s)
t
# ['s', 'p', 'a', 'm']

String to List of Words

Use split() to break a string into words: 

s = 'pining for the fjords'
t = s.split()
t
# ['pining', 'for', 'the', 'fjords']

# Split with custom delimiter
s = 'ex-parrot'
t = s.split('-')
t
# ['ex', 'parrot']

List to String

Use join() to combine list elements into a string: 

t = ['pining', 'for', 'the', 'fjords']

# Join with spaces
delimiter = ' '
s = delimiter.join(t)
s
# 'pining for the fjords'

# Join with no spaces
''.join(t)
# 'piningforthefjords'

# Join with custom delimiter
'-'.join(t)
# 'pining-for-the-fjords'

🔤 Important: join() is a string method, not a list method! Call it on the delimiter string and pass the list as an argument.

Looping Through Lists

For loops work beautifully with lists, allowing you to process each element in sequence. This is one of the most common patterns in Python programming.

Basic List Loop

Loop through each element in a list: 

cheeses = ['Cheddar', 'Edam', 'Gouda']

for cheese in cheeses:
    print(cheese)

# Output:
# Cheddar
# Edam
# Gouda

Combining split() and for

Process words from a sentence: 

s = 'pining for the fjords'

for word in s.split():
    print(word)

# Output:
# pining
# for
# the
# fjords

Empty List Behavior

A for loop over an empty list doesn't execute: 

for x in []:
    print('This never happens.')

# No output - loop body never runs

💡 Pattern: Use for loops to process each element of a list one at a time. This is cleaner and more readable than using indices!

Sorting Lists

Python's sorted() function makes it easy to arrange list elements in order. It works with numbers, strings, and more.

Sorting Without Modifying

sorted() returns a new sorted list, leaving the original unchanged: 

scramble = ['c', 'a', 'b']

# Get sorted version
sorted(scramble)
# ['a', 'b', 'c']

# Original is unchanged
scramble
# ['c', 'a', 'b']

Sorting Strings

sorted() works on any sequence, including strings: 

# Sort letters in a word
sorted('letters')
# ['e', 'e', 'l', 'r', 's', 't', 't']

# Convert back to string with join
''.join(sorted('letters'))
# 'eelrstt'

🎯 Use Case: Sorting letters is useful for finding anagrams—two words are anagrams if their sorted letters are identical!

Objects and Values

Understanding the difference between objects and values is crucial for working with lists. Two variables can refer to the same object or to different objects with the same value.

Same Object vs Same Value

Check if variables refer to the same object with the is operator: 

# Strings: Python optimizes by reusing objects
a = 'banana'
b = 'banana'
a is b
# True (same object!)

# Lists: Each assignment creates a new object
a = [1, 2, 3]
b = [1, 2, 3]
a is b
# False (different objects)

Equivalent vs Identical

Equivalent: Objects have the same value (use ==)

Identical: Variables refer to the same object (use is)

a = [1, 2, 3]
b = [1, 2, 3]

# Equivalent (same values)
a == b
# True

# Not identical (different objects)
a is b
# False

📊 Remember: If two objects are identical, they are also equivalent. But if they are equivalent, they are not necessarily identical!

Aliasing and References

When multiple variables refer to the same object, they are aliases. Changes made through one alias affect all others—this can cause surprising behavior if you're not careful!

Creating Aliases

When you assign one variable to another, both refer to the same object: 

a = [1, 2, 3]
b = a

# They are identical (same object)
b is a
# True

Aliasing Effects

Changes through one alias affect the other: 

a = [1, 2, 3]
b = a

# Modify through b
b[0] = 5

# a "sees" the change!
a
# [5, 2, 3]

Avoiding Aliasing Problems

Create a copy if you want independent lists: 

a = [1, 2, 3]
b = a[:]  # Create a copy

# Now they're different objects
b is a
# False

# Changes to b don't affect a
b[0] = 5
a
# [1, 2, 3]

Immutable Objects and Aliasing

With immutable objects like strings, aliasing isn't a problem: 

a = 'banana'
b = a

# You can't change a string anyway
# So it doesn't matter if they're the same object

⚠️ Danger Zone: Aliasing with mutable objects is error-prone. When working with lists, be careful about creating copies versus creating references!

Lists as Function Arguments

When you pass a list to a function, the function receives a reference to the original list—not a copy. This means functions can modify the caller's list!

Functions Modify Lists

Changes made inside a function persist after the function returns: 

def pop_first(lst):
    return lst.pop(0)

letters = ['a', 'b', 'c']

# Call the function
result = pop_first(letters)
result
# 'a'

# The original list is modified!
letters
# ['b', 'c']

Understanding the Reference

The parameter and the argument are aliases for the same list: 

# Inside the function:
# lst and letters refer to the same object
# So modifications through lst affect letters

🚨 Important: This is different from strings and numbers! When you pass a string or number to a function, the function can't change the caller's variable. But with lists, it can!

Building a Word List

Reading a file once and storing words in a list is much more efficient than reading the file multiple times. Here are two approaches.

Method 1: Loop and Append

Build the list one word at a time: 

word_list = []

for line in open('words.txt'):
    word = line.strip()
    word_list.append(word)

len(word_list)
# 113783

Method 2: Read and Split

Read the entire file and split it in one step: 

# Read entire file into a string
string = open('words.txt').read()

# Split into list of words
word_list = string.split()

len(word_list)
# 113783

Using the Word List

Now searching is fast with the in operator: 

'demotic' in word_list
# True

'contrafibularities' in word_list
# False

⚡ Efficiency: Loading the word list once lets you search it many times without re-reading the file. Much faster!

Debugging List Code

List methods behave differently than string methods. Understanding these differences helps avoid common bugs.

Common Mistake: Assigning Method Results

Most list methods modify the list and return None: 

# WRONG! remove returns None
t = [1, 2, 3]
t = t.remove(3)

# Now t is None
print(t)
# None

# Next operation fails
t.remove(2)
# AttributeError: 'NoneType' object has no attribute 'remove'

Correct Pattern

Don't assign the result—just call the method: 

# CORRECT: Just call the method
t = [1, 2, 3]
t.remove(3)

# t is still a list
print(t)
# [1, 2]

String Methods

Return new string, original unchanged

s = s.strip()

List Methods

Modify list in place, return None

lst.append(x)
