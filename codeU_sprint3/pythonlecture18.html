Defining Methods

Methods are functions defined inside a class. They make the connection between code and data explicit, creating more organized and intuitive programs.

From Function to Method

Move the function inside the class and change the parameter name to self: 

# Before: External function
def print_time(time):
    s = f'{time.hour:02d}:{time.minute:02d}:{time.second:02d}'
    print(s)

# After: Method inside class
class Time:
    """Represents the time of day."""
    
    def print_time(self):
        s = f'{self.hour:02d}:{self.minute:02d}:{self.second:02d}'
        print(s)

Two Ways to Call Methods

Function syntax vs. method syntax: 

start = Time()
start.hour = 9
start.minute = 40
start.second = 0

# Function syntax (uncommon)
Time.print_time(start)
# 09:40:00

# Method syntax (idiomatic!)
start.print_time()
# 09:40:00

The Receiver

The object a method is invoked on is called the receiver:

• In start.print_time(), start is the receiver

• Inside the method, self refers to the receiver

• Think of it as "sending a message" to an object

📌 Convention: Always name the first parameter of a method self. It's not required, but it's universal Python style!

Converting Functions to Methods

Let's convert more functions to methods. The pattern is simple: move inside the class, change the parameter name to self, and adjust attribute references.

Example: time_to_int

Convert time values to seconds: 

class Time:
    """Represents the time of day."""
    
    def time_to_int(self):
        minutes = self.hour * 60 + self.minute
        seconds = minutes * 60 + self.second
        return seconds

start = Time()
start.hour = 9
start.minute = 40
start.second = 0

start.time_to_int()
# 34800 (seconds since midnight)

Method vs Function Terminology

Functions

We call them

print_time(start)

Methods

We invoke them

start.print_time()

Note: "Call" and "invoke" mean the same thing—just different terminology!

Static Methods

Some methods don't need an instance to work. Static methods are invoked on the class itself, not on objects. They're perfect for factory functions!

The Chicken-and-Egg Problem

How do we create a Time object using a method if we need a Time object to call the method? 

class Time:
    """Represents the time of day."""
    
    @staticmethod
    def int_to_time(seconds):
        minute, second = divmod(seconds, 60)
        hour, minute = divmod(minute, 60)
        time = Time()
        time.hour = hour
        time.minute = minute
        time.second = second
        return time

# Call on the class, not an instance!
start = Time.int_to_time(34800)
print(f'{start.hour}:{start.minute}:{start.second}')
# 9:40:0

Static vs Instance Methods

Instance Method

• Has self parameter

• Invoked on objects

obj.method()

Static Method

• No self parameter

• Invoked on class

Class.method()

🏭 Use Case: Static methods are perfect for factory functions that create and return new instances!

Comparing Time Objects

Methods that compare objects use self for the first object and another parameter (often called other) for the second.

The is_after Method

Check if one time comes after another: 

class Time:
    """Represents the time of day."""
    
    def is_after(self, other):
        return self.time_to_int() > other.time_to_int()

start = Time()
start.hour = 9
start.minute = 40
start.second = 0

end = Time()
end.hour = 11
end.minute = 12
end.second = 0

end.is_after(start)
# True

# Reads like English: "end is after start?"

Elegant Syntax

Method syntax makes comparisons read naturally:

• end.is_after(start) reads like "end is after start?"

• Much clearer than is_after(end, start)

• Object-oriented code can be more intuitive!

Special Methods: __str__

Special methods have names surrounded by double underscores. They give objects custom behavior with Python's built-in functions and operators.

Converting Objects to Strings

Python calls __str__ when converting objects to strings: 

class Time:
    """Represents the time of day."""
    
    def __str__(self):
        s = f'{self.hour:02d}:{self.minute:02d}:{self.second:02d}'
        return s

end = Time()
end.hour = 11
end.minute = 12
end.second = 0

# Direct call
end.__str__()
# '11:12:00'

# Python calls it automatically
str(end)
# '11:12:00'

# Print calls str() automatically
print(end)
# 11:12:00

When __str__ is Called

• When you call str(object)

• When you print(object)

• When using objects in f-strings

• Anywhere Python needs a string representation

✨ Magic: Special methods let your objects work seamlessly with Python's built-in features!

The __init__ Method

The most important special method! __init__ initializes new objects automatically when they're created.

Automatic Initialization

Set attributes when creating objects: 

class Time:
    """Represents the time of day."""
    
    def __init__(self, hour=0, minute=0, second=0):
        self.hour = hour
        self.minute = minute
        self.second = second
    
    def __str__(self):
        return f'{self.hour:02d}:{self.minute:02d}:{self.second:02d}'

# Create and initialize in one step!
time = Time(9, 40, 0)
print(time)
# 09:40:00

Optional Parameters

Default values make parameters optional: 

# No arguments - all defaults
time = Time()
print(time)
# 00:00:00

# One argument - overrides hour
time = Time(9)
print(time)
# 09:00:00

# Two arguments - overrides hour and minute
time = Time(9, 45)
print(time)
# 09:45:00

# Three arguments - overrides all
time = Time(9, 45, 30)
print(time)
# 09:45:30

🎯 Best Practice: Always write __init__ and __str__ first when creating a new class. They make objects easier to create and debug!

Operator Overloading

Define special methods to make operators like + work with your custom types. This makes code more intuitive and Pythonic!

The __add__ Method

Make the + operator work with Time objects: 

class Time:
    def __add__(self, other):
        seconds = self.time_to_int() + other.time_to_int()
        return Time.int_to_time(seconds)

start = Time(9, 40)
duration = Time(1, 32)

# Use + operator!
end = start + duration
print(end)
# 11:12:00

What Happens Behind the Scenes

When you write start + duration:

1. Python calls start.__add__(duration)

2. __add__ converts both times to seconds

3. Adds the seconds together

4. Converts back to a Time object

5. Returns the new Time

Common Special Methods

Arithmetic

__add__ (+)

__sub__ (-)

__mul__ (*)

Comparison

__eq__ (==)

__lt__ (<)

__gt__ (>)

🎭 Operator Overloading: Changing operator behavior for custom types makes your objects feel like built-in Python types!

Debugging: Invariants

Invariants are conditions that should always be true. Writing methods to check invariants helps catch bugs early!

Time Invariants

What makes a valid Time object?

• Hour must be non-negative integer

• Minute must be 0-59 (integer)

• Second must be 0-59 (can have fractions)

• These conditions should ALWAYS be true

Validation Method

Check if a Time object is valid: 

class Time:
    def is_valid(self):
        if self.hour < 0 or self.minute < 0 or self.second < 0:
            return False
        if self.minute >= 60 or self.second >= 60:
            return False
        if not isinstance(self.hour, int):
            return False
        if not isinstance(self.minute, int):
            return False
        return True

time = Time(25, 70, -5)
time.is_valid()
# False

Using Assertions

Add checks at the start of methods: 

def is_after(self, other):
    assert self.is_valid(), 'self is not valid'
    assert other.is_valid(), 'other is not valid'
    return self.time_to_int() > other.time_to_int()

# Invalid time
bad_time = Time(0, 132, 0)
good_time = Time(9, 40, 0)

good_time.is_after(bad_time)
# AssertionError: other is not valid

🔍 Debugging Tip: Assertions help distinguish normal code from error-checking code. They fail fast when something goes wrong!
