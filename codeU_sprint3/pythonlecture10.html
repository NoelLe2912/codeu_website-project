Chapter 10: Dictionaries

Powerful Key-Value Mappings for Efficient Algorithms

Key-Value Pairs Hash Tables Fast Lookup

Dictionaries Are Mappings

A dictionary is like a list, but more powerful. Instead of using only integers as indices, dictionaries can use almost any type as a key. This makes them perfect for creating associations between related data.

The Limitation of Lists

With a list, you can only look up values by integer index: 

lst = ['zero', 'one', 'two']

# Look up by integer index
lst[1]
# 'one'

# But can't look up by word!
# How do we find which number "one" represents?

Creating a Dictionary

Use curly braces {} to create an empty dictionary, then add items: 

# Create empty dictionary
numbers = {}

# Add key-value pairs
numbers['zero'] = 0
numbers['one'] = 1
numbers['two'] = 2

numbers
# {'zero': 0, 'one': 1, 'two': 2}

Looking Up Values

Use the bracket operator with a key to get the corresponding value: 

numbers['two']
# 2

# If key doesn't exist, you get an error
numbers['three']
# KeyError: 'three'

Dictionary Properties

• Each item contains a key and a value, separated by a colon (:)

• Keys must be unique (no duplicates allowed)

• Use len() to count the number of items

• Keys can be almost any type (strings, numbers, tuples)

len(numbers)
# 3

💡 Key Concept: A dictionary represents a mapping from keys to values. Each key "maps to" its associated value, creating a powerful lookup system.

Different Ways to Create Dictionaries

Python provides multiple ways to create dictionaries. Choose the method that best fits your needs.

Method 1: All at Once

Create a dictionary with initial items using curly braces: 

numbers = {'zero': 0, 'one': 1, 'two': 2}
numbers
# {'zero': 0, 'one': 1, 'two': 2}

Method 2: Using dict()

The dict() function can create empty dictionaries or copies: 

# Create empty dictionary
empty = dict()
empty
# {}

# Make a copy of an existing dictionary
numbers_copy = dict(numbers)
numbers_copy
# {'zero': 0, 'one': 1, 'two': 2}

🔄 Best Practice: Create a copy before modifying a dictionary if you want to preserve the original. Changes to a copy don't affect the original.

Fast Lookups with in

The in operator works with dictionaries and is remarkably fast—even with millions of items! This is what makes dictionaries so powerful.

Checking for Keys

The in operator checks if something appears as a key: 

numbers = {'zero': 0, 'one': 1, 'two': 2}

# Check for key
'one' in numbers
# True

# Note: checks keys, not values!
1 in numbers
# False

Checking for Values

To check values, use the values() method: 

# Check if 1 is a value
1 in numbers.values()
# True

Why Dictionaries Are Fast

Dictionaries use hash tables—the in operator takes the same time regardless of size:

Lists: Slow

Searches from beginning to end

100,000 items = 100,000 checks

Dictionaries: Fast

Direct lookup using hash

100,000 items = ~1 check

Performance Comparison

Finding reversed words—list vs dictionary: 

# With a list: > 60 seconds (very slow!)
def too_slow():
    count = 0
    for word in word_list:
        if reverse_word(word) in word_list:
            count += 1
    return count

# With a dictionary: < 0.01 seconds (10,000x faster!)
def much_faster():
    count = 0
    for word in word_dict:
        if reverse_word(word) in word_dict:
            count += 1
    return count

⚡ Speed Boost: Converting a list to a dictionary for lookups can make your code thousands of times faster!

A Collection of Counters

Dictionaries are perfect for counting how many times things appear. This pattern—using a dictionary to count occurrences—is extremely common in data analysis.

Counting Pattern

For each item, if it's new, set count to 1. If it exists, increment the count: 

def value_counts(string):
    counter = {}
    for letter in string:
        if letter not in counter:
            counter[letter] = 1
        else:
            counter[letter] += 1
    return counter

counter = value_counts('brontosaurus')
counter
# {'b': 1, 'r': 2, 'o': 2, 'n': 1, 't': 1, 's': 2, 'a': 1, 'u': 2}

How It Works

Step 1: Start with an empty dictionary

Step 2: Loop through each letter

Step 3: If letter is new, add it with count 1

Step 4: If letter exists, increment its count

📊 Common Pattern: This counting technique works for anything—letters, words, items, events. It's one of the most useful dictionary patterns!

Looping Through Dictionaries

When you use a for loop with a dictionary, it automatically loops through the keys. You can also loop through values or key-value pairs.

Looping Through Keys

By default, for loops iterate over keys: 

counter = value_counts('banana')
counter
# {'b': 1, 'a': 3, 'n': 2}

for key in counter:
    print(key)

# Output:
# b
# a
# n

Looping Through Values

Use the values() method to loop through values: 

for value in counter.values():
    print(value)

# Output:
# 1
# 3
# 2

Looping Through Keys and Values

Loop through keys and look up each value: 

for key in counter:
    value = counter[key]
    print(key, value)

# Output:
# b 1
# a 3
# n 2

💡 Note: There's an even more concise way to loop through both keys and values using items()—you'll learn this in future chapters!

Lists and Dictionaries Together

You can use lists as dictionary values, but not as keys. Understanding why requires knowing about hash tables and immutability.

Lists as Values: Allowed

You can store lists as dictionary values: 

d = {4: ['r', 'o', 'u', 's']}
d
# {4: ['r', 'o', 'u', 's']}

Lists as Keys: Not Allowed

Attempting to use a list as a key causes a TypeError: 

letters = ['a', 'b', 'c', 'd']
d[letters] = 4
# TypeError: unhashable type: 'list'

Why Keys Must Be Hashable

• Dictionaries use hash functions to compute integer hash values from keys

• These hash values determine where to store items in memory

• If a key could change (is mutable), its hash value would change

• This would break the dictionary's internal organization

• Therefore, keys must be immutable (hashable)

✓ Hashable (Can Be Keys)

• Strings

• Numbers (int, float)

• Tuples (if they contain only hashable items)

✗ Not Hashable (Cannot Be Keys)

• Lists

• Dictionaries

• Sets

Building Lists with Accumulators

A common pattern is to loop through one collection while building another. The variable that collects items is called an accumulator.

Counting Palindromes

First, let's count how many palindromes exist: 

def is_palindrome(word):
    return reverse_word(word) == word

count = 0
for word in word_dict:
    if is_palindrome(word):
        count += 1

count
# 91

Collecting Palindromes

Now let's build a list of palindromes using an accumulator: 

palindromes = []

for word in word_dict:
    if is_palindrome(word):
        palindromes.append(word)

palindromes[:10]
# ['aa', 'aba', 'aga', 'aha', 'ala', 'alula', 'ama', 'ana', 'anna', 'ava']

Filtering Long Palindromes

Loop through the list again to select only long palindromes: 

long_palindromes = []

for word in palindromes:
    if len(word) >= 7:
        long_palindromes.append(word)

long_palindromes
# ['deified', 'halalah', 'reifier', 'repaper', 'reviver', 
#  'rotator', 'sememes']

🎯 Pattern Name: Looping through a collection and selecting some elements while omitting others is called filtering.

Memoization: Caching Results

Memoization uses a dictionary to store previously computed values, avoiding redundant calculations. This can make recursive functions dramatically faster.

The Problem: Slow Fibonacci

The standard recursive Fibonacci is very slow: 

def fibonacci(n):
    if n == 0:
        return 0
    if n == 1:
        return 1
    return fibonacci(n-1) + fibonacci(n-2)

# fibonacci(40) takes ~30 seconds!

Why It's Slow

The function recalculates the same values over and over:

• fibonacci(4) calls fibonacci(3) and fibonacci(2)

• fibonacci(3) calls fibonacci(2) and fibonacci(1)

• fibonacci(2) is computed multiple times!

• The redundancy grows exponentially

The Solution: Memoization

Store computed values in a dictionary: 

known = {0: 0, 1: 1}

def fibonacci_memo(n):
    if n in known:
        return known[n]
    
    res = fibonacci_memo(n-1) + fibonacci_memo(n-2)
    known[n] = res
    return res

# fibonacci_memo(40) takes ~0.00003 seconds!
# That's 1,000,000 times faster!

How Memoization Works

Step 1: Check if result is already in the dictionary

Step 2: If yes, return it immediately

Step 3: If no, compute the result

Step 4: Store the result in the dictionary

Step 5: Return the result

⚡ Amazing Speed Boost: Memoization can turn algorithms from impossibly slow to instant. It's one of the most powerful optimization techniques!

Debugging Large Datasets

When working with big dictionaries and datasets, traditional debugging techniques become unwieldy. Here are strategies that scale better.

Scale Down the Input

Work with smaller datasets while debugging:

• Start with just 10 lines instead of 100,000

• Find the smallest example that causes the error

• Gradually increase size as you fix bugs

Check Summaries and Types

Print summary information instead of entire contents: 

# Instead of printing entire dictionary
# print(huge_dict)  # Too much output!

# Print summaries
print(f"Dictionary has {len(huge_dict)} items")
print(f"First key: {list(huge_dict.keys())[0]}")
print(f"Type of values: {type(list(huge_dict.values())[0])}")

Write Self-Checks

Add sanity checks and consistency checks: 

# Sanity check
average = sum(numbers) / len(numbers)
assert average <= max(numbers), "Average too high!"
assert average >= min(numbers), "Average too low!"

# Consistency check
assert len(dict1) + len(dict2) == len(combined_dict)

🔧 Pro Tip: Time spent building good debugging infrastructure (scaffolding) reduces overall debugging time dramatically.
