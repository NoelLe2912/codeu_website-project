Integer Division and Modulus

The modulus operator (%) is more useful than it might seem at first. It returns the remainder when one number is divided by another.

Converting Minutes to Hours

Suppose a movie is 105 minutes long. How many hours and minutes is that? 

minutes = 105

# Integer division gives whole hours
hours = minutes // 60      # Result: 1

# Modulus gives remaining minutes
remainder = minutes % 60   # Result: 45

print(f"{hours} hours and {remainder} minutes")

Checking Divisibility

If x % y is zero, then x is divisible by y

10 % 2 # 0 (divisible)
10 % 3 # 1 (not divisible)

Extracting Digits

Get the rightmost digits of a number

123 % 10 # 3 (last digit)
123 % 100 # 23 (last 2 digits)

Clock Arithmetic

If an event starts at 11 AM and lasts 3 hours, when does it end? 

start = 11
duration = 3
end = (start + duration) % 12
# Result: 2 (2 PM)

Boolean Expressions

A boolean expression is one that evaluates to either True or False. These are fundamental for making decisions in your programs.

Comparison Operators

x = 5
y = 7

x == y    # False (equal to)
x != y    # True (not equal to)
x > y     # False (greater than)
x < y     # True (less than)
x >= y    # False (greater than or equal)
x <= y    # True (less than or equal)

⚠️ Common Mistake: Don't confuse = (assignment) with == (comparison)!

x = 5 # Assigns 5 to x
x == 5 # Checks if x equals 5

The bool Type

True and False are special values of type bool: 

type(True)   # 
type(False)  # 

# They're not strings!
type("True")  #

Logical Operators

Logical operators (and, or, not) let you combine boolean expressions to create more complex conditions.

and

Both conditions must be True

x > 0 and x < 10

True only if x is between 0 and 10

or

At least one must be True

x % 2 == 0 or x % 3 == 0

True if divisible by 2 or 3

not

Negates the condition

not x > y

True if x is NOT greater than y

Combining Conditions

# Check if a number is in a range
age = 25
if age >= 18 and age < 65:
    print("Working age")

# Check multiple conditions
score = 85
if score >= 90 or score < 60:
    print("Extreme score")

# Negate a condition
is_raining = False
if not is_raining:
    print("Let's go outside!")

If Statements

Conditional statements let you execute different code based on whether conditions are true or false. This is how programs make decisions.

Basic If Statement

The simplest form executes code only if a condition is true: 

x = 10

if x > 0:
    print('x is positive')
    print('This runs too!')

# Both lines execute because x > 0 is True

The else Clause

Use else to specify what happens when the condition is false: 

x = 7

if x % 2 == 0:
    print('x is even')
else:
    print('x is odd')

# Output: x is odd

Chained Conditionals

Use elif (else if) to check multiple conditions: 

x = 5
y = 7

if x < y:
    print('x is less than y')
elif x > y:
    print('x is greater than y')
else:
    print('x and y are equal')

# Output: x is less than y

💡 Key Point: Only ONE branch executes. Python checks conditions in order and runs the first one that's true, then skips the rest.

Nested Conditionals

You can put if statements inside other if statements, but this can get confusing. There are usually better alternatives.

❌ Nested (Harder to Read)

if x == y:
    print('equal')
else:
    if x < y:
        print('less')
    else:
        print('greater')

✓ Chained (Easier to Read)

if x == y:
    print('equal')
elif x < y:
    print('less')
else:
    print('greater')

Using Logical Operators

You can often simplify nested conditionals with logical operators:

Nested:

if 0 < x:
    if x < 10:
        print('single digit')

Better:

if 0 < x < 10:
    print('single digit')

Recursion: Functions Calling Themselves

A function can call itself! This is called recursion, and it's one of the most powerful ideas in programming. It might seem strange at first, but it's incredibly useful for certain problems.

Simple Example: Countdown

Here's a function that counts down from n to 0: 

def countdown(n):
    if n <= 0:
        print('Blastoff!')
    else:
        print(n)
        countdown(n - 1)  # Function calls itself!

countdown(3)
# Output:
# 3
# 2
# 1
# Blastoff!

How It Works

Step 1: countdown(3) prints 3, then calls countdown(2)

Step 2: countdown(2) prints 2, then calls countdown(1)

Step 3: countdown(1) prints 1, then calls countdown(0)

Step 4: countdown(0) prints "Blastoff!" and returns

Step 5: Each function returns in reverse order

Another Example: Print N Times

def print_n_times(message, n):
    if n > 0:
        print(message)
        print_n_times(message, n - 1)

print_n_times('Hello!', 4)
# Output:
# Hello!
# Hello!
# Hello!
# Hello!

🔑 Key Concept: Every recursive function needs a BASE CASE (a condition that stops the recursion). Without it, the function will call itself forever!

Understanding Recursive Calls

Each time a function is called, Python creates a new frame on the stack. With recursion, you get multiple frames for the same function with different parameter values.

Stack Diagram for countdown(3)

countdown

n → 0 (BASE CASE)

countdown

n → 1

countdown

n → 2

countdown

n → 3

__main__

The Base Case

The bottom frame where n=0 is the base case. It doesn't make another recursive call, so the stack stops growing and functions start returning.

Infinite Recursion

If a recursive function never reaches its base case, it will keep calling itself forever until Python runs out of stack space. This is called infinite recursion.

Example of Infinite Recursion

def recurse():
    recurse()  # No base case!

# This will crash:
# recurse()
# RecursionError: maximum recursion depth exceeded

Common Mistake: Wrong Base Case

# This looks OK but has a bug!
def countdown_by_two(n):
    if n == 0:
        print('Blastoff!')
    else:
        print(n)
        countdown_by_two(n - 2)

# Works fine:
countdown_by_two(6)  # 6, 4, 2, Blastoff!

# But this causes infinite recursion:
# countdown_by_two(5)  # 5, 3, 1, -1, -3, ...

🐛 Debugging Tip: Always check:

  • • Does your function have a base case?
  • • Does the base case actually stop the recursion?
  • • Will you always reach the base case?

Getting Input from Users

The input() function lets your programs interact with users by asking for information. Note: In the terminal on this page, input() won't work, but it works in regular Python!

Basic Input

# Ask for user's name
name = input('What is your name? ')
print('Hello,', name)

# Ask for a number (returns a string!)
age = input('How old are you? ')
age = int(age)  # Convert to integer
print('Next year you will be', age + 1)

⚠️ Important: input() always returns a string, even if the user types a number. Use int() or float() to convert when needed.

Practical Examples

Let's look at some practical applications of conditionals and recursion.

Example: Factorial Function

The factorial of n (written n!) is n × (n-1) × (n-2) × ... × 1. This is perfect for recursion: 

def factorial(n):
    """Calculate n! recursively."""
    if n <= 1:
        return 1
    else:
        return n * factorial(n - 1)

# Test it
print(factorial(5))  # 5 × 4 × 3 × 2 × 1 = 120
print(factorial(3))  # 3 × 2 × 1 = 6

Example: Grade Calculator

A practical use of chained conditionals: 

def get_grade(score):
    """Convert numeric score to letter grade."""
    if score >= 90:
        return 'A'
    elif score >= 80:
        return 'B'
    elif score >= 70:
        return 'C'
    elif score >= 60:
        return 'D'
    else:
        return 'F'

print(get_grade(85))  # B
print(get_grade(92))  # A

Example: Fibonacci Numbers

The Fibonacci sequence: each number is the sum of the previous two: 

def fibonacci(n):
    """Return the nth Fibonacci number."""
    if n <= 1:
        return n
    else:
        return fibonacci(n - 1) + fibonacci(n - 2)

# First 8 Fibonacci numbers
for i in range(8):
    print(fibonacci(i), end=' ')
# Output: 0 1 1 2 3 5 8 13

Debugging Tips

Conditionals and recursion can introduce subtle bugs. Here's how to catch them.

Common Conditional Errors

  • • Using = instead of ==
  • • Forgetting the colon after if/elif/else
  • • Wrong indentation in blocks
  • • Checking conditions in wrong order
  • • Forgetting parentheses in complex conditions

Common Recursion Errors

  • • Missing base case
  • • Base case never reached
  • • Not moving toward base case
  • • Forgetting to return value
  • • Too many recursive calls (stack overflow)

Testing Recursive Functions

Always test with:

  • • Base case: Does it work when it should stop?
  • • Small values: Does it work for n=1, n=2?
  • • Edge cases: What about n=0 or negative numbers?
  • • Typical cases: Does it work for normal inputs?
