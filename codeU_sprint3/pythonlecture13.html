Filenames and Paths

Before working with files, you need to locate them. Files are organized in directories (folders), and paths specify where files live in the filesystem hierarchy.

Understanding Paths

Paths identify the location of files and directories: 

# Conceptual example (won't run in browser terminal)
# import os

# Get current directory
# os.getcwd()
# '/home/username'

# Absolute path (starts from root)
# '/home/username/documents/memo.txt'

# Relative path (relative to current directory)
# 'documents/memo.txt'

Path Types

Absolute Path

Complete location from root

/home/user/file.txt

Relative Path

Location from current directory

documents/file.txt

Path Operations (Conceptual)

Common operations for working with paths: 

# Join paths (works on any OS)
# os.path.join('photos', 'jan-2023', 'photo1.jpg')
# 'photos/jan-2023/photo1.jpg'

# Check if path exists
# os.path.exists('photos')
# True

# Check if it's a file
# os.path.isfile('photo.jpg')
# True

# Check if it's a directory
# os.path.isdir('photos')
# True

⚠️ Platform Differences: Windows uses backslashes (\) while Mac/Linux use forward slashes (/). Use os.path.join() to write cross-platform code!

F-Strings for Formatting

F-strings let you embed Python expressions directly in strings, making it easy to create formatted text for writing to files or displaying data.

Basic F-String Syntax

Prefix strings with 'f' and use curly braces for expressions: 

num_camels = 23
num_years = 1.5

# Simple variable substitution
f'I have spotted {num_camels} camels'
# 'I have spotted 23 camels'

# Multiple expressions
f'In {num_years} years I have spotted {num_camels} camels'
# 'In 1.5 years I have spotted 23 camels'

F-Strings with Expressions

Include calculations and function calls: 

# Expressions and function calls
line = f'In {round(num_years * 12)} months I spotted {num_camels} camels'
line
# 'In 18 months I spotted 23 camels'

# Can include any data type
t = [1, 2, 3]
d = {'one': 1}
f'List: {t}, Dict: {d}'
# "List: [1, 2, 3], Dict: {'one': 1}"

Writing to Files (Conceptual)

F-strings make file writing clean and readable: 

# Conceptual example
# writer = open('data.txt', 'w')
# writer.write(f'Years: {num_years}\n')
# writer.write(f'Camels: {num_camels}\n')
# writer.close()

# Result in file:
# Years: 1.5
# Camels: 23

💡 Pro Tip: F-strings are cleaner and faster than older string formatting methods like % or .format()!

Serialization: Saving Data Structures

Serialization converts complex data structures (like dictionaries) into strings that can be saved to files. Deserialization converts them back.

Why Serialize?

Save configuration data, user preferences, or application state: 

config = {
    'photo_dir': 'photos',
    'data_dir': 'photo_info',
    'extensions': ['jpg', 'jpeg'],
}

# This dictionary needs to be saved to a file
# How do we convert it to text?

YAML Format (Conceptual)

YAML is human-readable and easy for programs to parse: 

# Conceptual YAML example
# import yaml

# Write to file
# writer = open('config.yaml', 'w')
# yaml.dump(config, writer)
# writer.close()

# YAML format looks like:
# data_dir: photo_info
# extensions:
# - jpg
# - jpeg
# photo_dir: photos

# Read back
# reader = open('config.yaml')
# config_readback = yaml.safe_load(reader)
# config_readback
# {'data_dir': 'photo_info', ...}

Key Terms

Serialization: Object → String (for saving)

Deserialization: String → Object (for loading)

Round-trip: Serialize then deserialize = equivalent object

📦 Common Formats: JSON, YAML, XML, and Python's pickle are all serialization formats with different trade-offs.

Databases: Persistent Key-Value Storage

Databases store data more efficiently than text files. Python's shelve module provides a simple dictionary-like database called a key-value store.

What is a Database?

A file organized for efficient data operations:

• More efficient than reading entire text files

• Can handle large amounts of data

• Supports fast lookups and updates

• Provides data persistence between program runs

Shelve: Dictionary-Like Database

Shelve objects work like dictionaries but save to disk: 

# Conceptual example
# import shelve

# Open a shelf (creates if needed)
# db = shelve.open('captions', 'c')

# Use like a dictionary
# db['jan-2023/photo1.jpg'] = 'Cat nose'
# db['jan-2023/photo2.jpg'] = 'Sunset'

# Retrieve values
# db['jan-2023/photo1.jpg']
# 'Cat nose'

# Update values
# db['jan-2023/photo1.jpg'] = 'Close up of cat nose'

# Close when done
# db.close()

Dictionary Methods Work Too

Shelves support most dictionary operations: 

# Check if key exists
# 'jan-2023/photo1.jpg' in db
# True

# Get keys and values
# list(db.keys())
# ['jan-2023/photo1.jpg', 'jan-2023/photo2.jpg']

# Loop through
# for key in db:
#     print(key, ':', db[key])

⚠️ Important: Shelves create multiple files (.dat for data, .dir for directory info). Always close shelves when done!

Storing Data Structures in Shelves

Shelves can store complex data structures like lists and dictionaries as values. However, there's a tricky gotcha when updating mutable values!

Lists as Values

Store lists of related items: 

# Example: anagram database
# def sort_word(word):
#     return ''.join(sorted(word))

# db = shelve.open('anagrams', 'n')

# Store a list
# key = sort_word('pots')  # 'opst'
# db[key] = ['pots']
# db[key]
# ['pots']

The Update Gotcha!

You can't modify shelf values in place: 

# WRONG - doesn't update the shelf!
# db[key].append('tops')
# db[key]
# ['pots']  # Still only one word!

# The problem: Getting the value creates a copy
# Modifying the copy doesn't affect the shelf

The Correct Way

Read, modify, write back: 

# CORRECT - read, modify, write
# anagram_list = db[key]      # Read
# anagram_list.append('tops')  # Modify
# db[key] = anagram_list       # Write back

# db[key]
# ['pots', 'tops']  # Now it works!

🔑 Remember: Always use the three-step pattern: read the value, modify it, write it back. Direct modification doesn't work!

Hash Functions and File Comparison

Hash functions convert data into fixed-size digests. Files with identical content produce identical digests, making it efficient to find duplicates.

Why Use Hash Functions?

Comparing file contents directly is slow for many files:

Slow way: Compare every pair of files (N² comparisons)

Fast way: Compute digest for each file, compare digests

• Same content → Same digest

• Different content → Different digest (almost always)

MD5 Hash Example (Conceptual)

MD5 produces a 128-bit digest from any amount of data: 

# Conceptual example
# import hashlib

# def md5_digest(filename):
#     data = open(filename, 'rb').read()
#     md5_hash = hashlib.md5()
#     md5_hash.update(data)
#     return md5_hash.hexdigest()

# Two different files
# md5_digest('photo1.jpg')
# 'aa1d2fc25b7ae247b2931f5a0882fa37'

# md5_digest('photo2.jpg')
# '6a501b11b01f89af9c3f6591d7f02c49'

# Different content = different digests!

Finding Duplicates

Map digests to file paths to find duplicates: 

# Strategy:
# 1. For each file, compute MD5 digest
# 2. Store in shelf: digest → list of paths
# 3. Any digest with 2+ paths = duplicate files!

# digest_db[digest1] = ['path/to/file1.jpg']
# digest_db[digest2] = ['path/to/file2.jpg', 'path/to/copy.jpg']
#                       ^^^ These are duplicates!

🔐 Real World: Hash functions are also used for passwords, digital signatures, and detecting data corruption!

Recursively Searching Directories

To find all files in a directory tree, we need to search recursively through subdirectories. This is a classic application of recursion!

The Walk Function (Conceptual)

Recursively visit every file and subdirectory: 

# Conceptual recursive directory walker
# def walk(dirname):
#     for name in os.listdir(dirname):
#         path = os.path.join(dirname, name)
#         
#         if os.path.isfile(path):
#             print(path)  # Found a file
#         elif os.path.isdir(path):
#             walk(path)   # Recurse into subdirectory

# walk('photos')
# Output:
# photos/notes.txt
# photos/jan-2023/photo1.jpg
# photos/jan-2023/photo2.jpg
# photos/feb-2023/photo1.jpg

How It Works

Step 1: List all items in the current directory

Step 2: For each item, check if it's a file or directory

Step 3: If file → process it (print, hash, etc.)

Step 4: If directory → call walk() on it (recursion!)

Ephemeral vs Persistent Programs

Understanding the difference between temporary and long-lasting data is crucial for designing robust programs.

Ephemeral Programs

• Run for a short time

• Data exists only in memory

• Everything lost when program ends

• Start fresh each run

Examples: calculators, converters, simple scripts

Persistent Programs

• Run indefinitely or frequently

• Save data to disk

• Remember state between runs

• Pick up where they left off

Examples: databases, web servers, games with save files

Persistence Mechanisms

Text files: Simple, human-readable, slow for large data

Databases: Fast, efficient, supports complex queries

Serialized objects: Save entire data structures

Cloud storage: Remote, accessible anywhere

💾 Design Choice: Choose persistence based on your needs. Games save player progress. Web apps save user data. Calculators don't need to save anything!

Debugging File Operations

File operations introduce new debugging challenges—invisible whitespace, platform differences, and permission errors.

Invisible Whitespace Problems

Use repr() to see hidden characters: 

s = '1 2\t 3\n 4'

# Printing hides whitespace
print(s)
# 1 2	 3
#  4

# repr() shows everything
print(repr(s))
# '1 2\t 3\n 4'

Common File Issues

• Line endings: \n (Unix), \r\n (Windows), \r (old Mac)

• File encoding: UTF-8, ASCII, Latin-1 incompatibilities

• Case sensitivity: Unix is case-sensitive, Windows often isn't

• Path separators: / vs \ on different platforms

• Permissions: Can't read/write without proper access

🔍 Debugging Tip: When file operations fail mysteriously, check the path, permissions, and use repr() to inspect strings for hidden characters!
