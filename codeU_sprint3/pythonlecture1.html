Working with Numbers: Arithmetic Operators

Python uses special symbols called operators to perform calculations. Think of these as the mathematical tools in your programming toolbox. Each operator tells Python what kind of calculation to perform.

Basic Mathematical Operations

Addition (+)

Combines two numbers together

25 + 17 # Result: 42

Subtraction (-)

Finds the difference between numbers

50 - 8 # Result: 42

Multiplication (*)

Multiplies numbers together

6 * 7 # Result: 42

Division (/)

Divides one number by another

84 / 2 # Result: 42.0

⚡ Key Insight: Notice that division always gives you a decimal number (42.0), even when dividing evenly. This is called a floating-point number in Python!

Advanced Operators

Integer Division (//)

Divides and rounds down to a whole number

85 // 2 # Result: 42 (not 42.5!)

Exponentiation (**)

Raises a number to a power

2 ** 8 # Result: 256

Understanding Expressions

An expression is any combination of numbers and operators that Python can calculate. Think of it like a math problem that Python solves for you.

Order Matters!

Just like in math class, Python follows specific rules about which operations to do first. This is called the order of operations.

# Exponents happen first

5 + 3 ** 2  # 3**2 is calculated first: 5 + 9 = 14

# Multiplication before addition

10 + 4 * 3  # 4*3 is calculated first: 10 + 12 = 22

# Use parentheses to change the order

(10 + 4) * 3  # Parentheses first: 14 * 3 = 42
1️⃣

Parentheses First

Calculate anything in ( ) before other operations

2️⃣

Exponents Second

Powers and roots come next

3️⃣

Then × and ÷

Finally + and - from left to right

Python's Built-in Functions

Functions are pre-written pieces of code that perform specific tasks. You call a function by writing its name followed by parentheses containing the value you want to work with.

Common Number Functions

# Round to the nearest whole number

round(7.8) # Result: 8
round(7.2) # Result: 7

# Get the absolute value (distance from zero)

abs(-15) # Result: 15
abs(15) # Result: 15

⚠️ Common Mistake: Function calls MUST include parentheses. Writing abs 15 will cause a syntax error. Always use abs(15) instead!

Strings: Working with Text

Strings are sequences of characters (letters, numbers, symbols) wrapped in quotation marks. They let Python work with text, not just numbers.

Creating Strings

# Single quotes work
'Python is awesome!'

# Double quotes work too
"Learning to code"

# Use double quotes for apostrophes
"I'm learning Python today"

String Operations

# Concatenation: Joining strings together

'Hello, ' + 'world!'  # Result: 'Hello, world!'

# Repetition: Repeating strings

'Ha' * 3  # Result: 'HaHaHa'

# Length: Counting characters

len('Python')  # Result: 6

💡 Remember: The + operator has different meanings for numbers and strings. With numbers it adds, with strings it joins them together!

Understanding Data Types

Every value in Python belongs to a specific category called a type. The three fundamental types you'll work with are integers, floating-point numbers, and strings.

🔢

int

Whole numbers without decimals

42, -7, 1000
🎯

float

Numbers with decimal points

3.14, -0.5, 42.0
📝

str

Text in quotation marks

'Hello', "123"

Checking and Converting Types

# Use type() to check what type a value is

type(42)      # Result: <class 'int'>
type(42.0)    # Result: <class 'float'>
type('42')    # Result: <class 'str'>

# Convert between types

int(3.9)      # Result: 3 (rounds down)
float(42)     # Result: 42.0
int('100')    # Result: 100

⚠️ Watch Out: Numbers in quotes are strings, not numbers! '123' is text, while 123 is a number. You can't do math with strings unless you convert them first.

Programming Languages vs Human Languages

Programming languages like Python are very different from the languages we speak every day. Understanding these differences will help you become a better programmer.

Key Differences

Precision vs Ambiguity

In English, "I saw her duck" could mean multiple things. In Python, every instruction has exactly one meaning. There's no room for interpretation.

Conciseness vs Redundancy

We often repeat ourselves when speaking to ensure understanding. Python code is dense and precise—every character matters.

Literal vs Figurative

If you tell someone "break a leg," they know you're wishing them luck. If you tell Python to break something, it will try to actually break it!

📚 Learning Tip: Reading code takes practice and patience. Don't read it like a novel—study the structure, check every detail, and take your time. Small mistakes in spelling or punctuation can completely change what a program does!
