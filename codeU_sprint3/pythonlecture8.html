Strings Are Sequences

Unlike integers or booleans, strings are sequences—collections of individual characters in a specific order. This means you can access each character separately using its position.

Accessing Characters with Indexing

Use the bracket operator [] with an index to select individual characters: 

fruit = 'banana'
letter = fruit[1]
letter
# 'a'

# The first character has index 0
fruit[0]
# 'b'

Zero-Based Indexing

Python counts from 0, not 1! The index is an offset from the beginning:

b
index 0
a
index 1
n
index 2
a
index 3
n
index 4
a
index 5

Using Variables and Expressions

Indices can be variables or expressions: 

i = 1
fruit[i]
# 'a'

fruit[i + 1]
# 'n'

# But indices must be integers
fruit[1.5]
# TypeError: string indices must be integers

Negative Indices

Use negative indices to count backward from the end: 

fruit = 'banana'

# Get the last character
fruit[-1]
# 'a'

# Second to last
fruit[-2]
# 'n'

# This avoids out-of-range errors
n = len(fruit)
fruit[n-1]  # Works, but more complicated
fruit[-1]   # Easier!

⚠️ Common Error: Don't forget that indices start at 0! If a string has 6 characters, valid indices are 0-5, not 1-6. Using fruit[6] causes an IndexError.

Extracting String Slices

A slice is a segment of a string. You can extract slices using the [n:m] operator, which returns characters from position n up to (but not including) position m.

Basic Slicing

The slice [n:m] includes the character at index n but excludes the character at index m: 

fruit = 'banana'
fruit[0:3]
# 'ban'

fruit[3:6]
# 'ana'

Understanding Slice Boundaries

Think of indices as pointing between characters, not at them:

0
b
1
a
2
n
3
a
4
n
5
a
6

fruit[0:3] selects 'ban' (positions 0, 1, 2)

Omitting Indices

You can omit the start or end index for convenient shortcuts: 

fruit = 'banana'

# Omit first index: start from beginning
fruit[:3]
# 'ban'

# Omit second index: go to the end
fruit[3:]
# 'ana'

# What does fruit[:] mean?
fruit[:]
# 'banana' (entire string!)

Empty Slices

If the first index is greater than or equal to the second, you get an empty string: 

fruit[3:3]
# ''

len(fruit[3:3])
# 0

Immutability of Strings

Strings are immutable, which means you cannot change individual characters. Once a string is created, it cannot be modified—you can only create new strings.

What Happens When You Try to Modify?

Attempting to change a character causes a TypeError: 

greeting = 'Hello, world!'
greeting[0] = 'J'
# TypeError: 'str' object does not support item assignment

Creating Modified Copies

Instead of modifying, create a new string using concatenation and slicing: 

greeting = 'Hello, world!'

# Create a new string with different first letter
new_greeting = 'J' + greeting[1:]
new_greeting
# 'Jello, world!'

# Original string is unchanged
greeting
# 'Hello, world!'

💡 Key Concept: Immutability means strings are safe to share and use in multiple places. You never have to worry about one part of your program changing a string that another part is using.

Comparing Strings

Relational operators work with strings to check equality and alphabetical order. This is useful for sorting and searching operations.

Equality Testing

Use == to check if two strings are identical: 

word = 'banana'

if word == 'banana':
    print('All right, banana.')

# Output: All right, banana.

Alphabetical Ordering

Use < and > to compare strings alphabetically: 

def compare_word(word):
    if word < 'banana':
        print(word, 'comes before banana.')
    elif word > 'banana':
        print(word, 'comes after banana.')
    else:
        print('All right, banana.')

compare_word('apple')
# apple comes before banana.

compare_word('cherry')
# cherry comes after banana.

Case Sensitivity Warning

Python treats uppercase and lowercase differently—all uppercase letters come before lowercase: 

compare_word('Pineapple')
# Pineapple comes before banana.

# To fix this, convert to lowercase first
def compare_word_fixed(word):
    word_lower = word.lower()
    if word_lower < 'banana':
        print(word, 'comes before banana.')
    elif word_lower > 'banana':
        print(word, 'comes after banana.')
    else:
        print('All right, banana.')

🔤 Pro Tip: When comparing strings for ordering, always convert them to a standard format (usually lowercase) to avoid unexpected results from case differences.

Useful String Methods

Strings come with many built-in methods that perform common operations. Methods are functions associated with objects, called using the dot operator.

Case Conversion Methods

Convert between uppercase and lowercase: 

word = 'banana'

# Convert to uppercase
new_word = word.upper()
new_word
# 'BANANA'

# Convert to lowercase
mixed = 'BaNaNa'
mixed.lower()
# 'banana'

Testing String Content

Check what a string starts or ends with: 

line = '*** START OF BOOK ***'

# Check if it starts with a pattern
line.startswith('***')
# True

line.startswith('Chapter')
# False

# Check if it ends with a pattern
filename = 'document.txt'
filename.endswith('.txt')
# True

Method Syntax

Methods use the dot operator:

string.method()

Function Syntax

Functions use parentheses:

function(string)

📚 Terminology: When you call a method on an object (like word.upper()), this is called an invocation. You are "invoking" the upper method on word.

Reading and Writing Files

String methods are essential when working with text files. Let's learn how to read from files, process the text, and write modified versions.

Opening Files for Reading

Use the open function to create a file object: 

reader = open('book.txt')

# Loop through lines
for line in reader:
    print(line.strip())

Opening Files for Writing

Add 'w' mode to open a file for writing: 

reader = open('input.txt')
writer = open('output.txt', 'w')

# Process and write lines
for line in reader:
    # Modify the line somehow
    writer.write(line)

# Always close files when done
reader.close()
writer.close()

Using the break Statement

Exit a loop early when you find what you're looking for: 

def is_special_line(line):
    return line.startswith('*** ')

# Find the first special line
for line in reader:
    if is_special_line(line):
        break  # Exit the loop immediately

# 'line' now contains the special line we found

⚠️ Important: Opening a file with 'w' mode will erase any existing content! If the file exists, it will be overwritten. Always be careful when writing files.

Finding and Replacing Text

String methods make it easy to search for patterns and replace them with new text. These operations are fundamental to text processing.

Using the in Operator

Check if a substring appears anywhere in a string: 

line = "Jonathan Harker's Journal"

if 'Jonathan' in line:
    print('Found Jonathan!')

# Output: Found Jonathan!

Counting Occurrences

Use the count method to find how many times a substring appears: 

text = "banana"
text.count('a')
# 3

text.count('na')
# 2

# Count across entire file
total = 0
for line in open('book.txt'):
    total += line.count('Jonathan')

print(f'Found {total} occurrences')

The replace Method

Replace all occurrences of a substring with another: 

line = "Hello Jonathan, how are you Jonathan?"

# Replace all occurrences
new_line = line.replace('Jonathan', 'Thomas')
new_line
# "Hello Thomas, how are you Thomas?"

# Process entire file
writer = open('output.txt', 'w')
for line in open('input.txt'):
    line = line.replace('Jonathan', 'Thomas')
    writer.write(line)
writer.close()

🔍 Remember: String methods like replace create new strings—they don't modify the original. This is because strings are immutable!

Introduction to Regular Expressions

Regular expressions (regex) are powerful patterns that let you search for complex text patterns. They go far beyond simple substring matching.

Basic Pattern Matching

Import the re module and use search to find patterns: 

import re

text = "I am Dracula; and I bid you welcome, Mr. Harker."
pattern = 'Dracula'

result = re.search(pattern, text)
result
# <re.Match object; span=(5, 12), match='Dracula'>

Working with Match Objects

When search finds a match, it returns a Match object with useful information: 

# Get the matched text
result.group()
# 'Dracula'

# Get the position (start, end)
result.span()
# (5, 12)

# Get the original string
result.string
# 'I am Dracula; and I bid you welcome, Mr. Harker.'

Handling No Match

If the pattern isn't found, search returns None: 

result = re.search('Count', text)
print(result)
# None

# Check before using the result
if result != None:
    print(result.group())
else:
    print('Pattern not found')

💡 Starting Simple: For now, we're using regex just like the in operator. The real power comes with special characters that let you match complex patterns!

Powerful Regex Patterns

Regular expressions use special characters to create flexible patterns. These let you search for variations and alternatives that would be tedious with basic string methods.

The OR Operator (|)

Use | to match either one pattern or another: 

import re

# Match either "Mina" or "Murray"
pattern = 'Mina|Murray'
text = 'CHAPTER V. Letters—Lucy and Mina'

result = re.search(pattern, text)
result.group()
# 'Mina'

Anchors: Start (^) and End ($)

Use ^ to match the beginning and $ to match the end of a line: 

# Match lines starting with "Chapter"
pattern = '^Chapter'
result = re.search(pattern, 'Chapter 1. Introduction')
result.group()
# 'Chapter'

# Match lines ending with "END"
pattern = 'END$'
result = re.search(pattern, 'THE END')
result.group()
# 'END'

Grouping with Parentheses

Use parentheses to group parts of a pattern: 

# Match "center" or "centre" (British spelling)
pattern = 'cent(er|re)'

text = 'the centre of the room'
result = re.search(pattern, text)
result.group()
# 'centre'

Optional Characters (?)

Use ? to make the previous character optional: 

# Match "color" or "colour"
pattern = 'colou?r'

# Matches with 'u'
re.search(pattern, 'colour').group()
# 'colour'

# Matches without 'u'
re.search(pattern, 'color').group()
# 'color'

Common Regex Special Characters

|

OR - match either pattern

^

Start of string

$

End of string

?

Optional (0 or 1 times)

()

Grouping

.

Any character

String Substitution with Regex

The re.sub function performs find-and-replace operations using regex patterns, making it more powerful than the basic replace method.

Basic Substitution

Use re.sub to replace patterns with new text: 

import re

pattern = 'colou?r'
text = 'The colour of the coloured fabric'

# Replace all matches
result = re.sub(pattern, 'color', text)
result
# 'The color of the colored fabric'

Processing Entire Files

Combine regex substitution with file processing: 

import re

# Convert British to American spelling
pattern = 'colou?r'
writer = open('output.txt', 'w')

for line in open('input.txt'):
    line = re.sub(pattern, 'color', line)
    writer.write(line)

writer.close()

🎯 Practical Use: Regex substitution is perfect for standardizing text, fixing formatting issues, or converting between variations (like British vs American spelling).

Real-World Application: Wordle

Wordle is a popular word game where you guess a five-letter word in six attempts. After each guess, you learn which letters are in the word and which are in the correct position.

How Wordle Works

• You have six attempts to guess a five-letter word

• Each guess must be a valid English word

• After each guess, letters are colored:

Green - Letter is correct and in the right position

Yellow - Letter is in the word but wrong position

Gray - Letter is not in the word

Example Game

Target word: MOWER

Guess 1: TRIED

T
R
I
E
D

E is in position 5! R is in the word but wrong spot.
